GET-FIELD-VALUE()

Description
Returns the current value of the field whose name is specified in the string expression. Note that the field name here is considered data, so it must either be contained in a data field or specified as a quoted alpha-literal. This function always returns the character representation of the field, regardless of its type.
Syntax
GET-FIELD-VALUE(string-expression)
Category
Environment
Runtime version
6.4 and above

Notes

The GET-FIELD-VALUE function requires some preparation before it can be used. This includes ensuring the following:

  • Objects required are known at compile time.

  • The required record is current.

  • The required field on that record is referenced before calling get-field-value (see example below).

  • Array fields should be properly referenced by a subscript. If no subscript is passed, the first element’s value is returned. If an invalid subscript is passed, SPACES is returned.

  • Base type of the field (numeric or string) can be obtained from the dictionary to determine which function to use. This is optional as the GET-FIELD-VALUE function can be used to obtain a character representation of any type of field.

Without the above preparation, the only fields guaranteed to return values are the key fields of the current objects referenced by the program.

Referencing a field

It is not enough to only have currency on a record. You must also have referenced the field required (that is, set or printed it), in order for the GET-FIELD-VALUE to return values correctly.

Two methods can be used to reference all fields on a record simultaneously:

  • if 1 = 0 insert deb-master endif (sets references to all fields but never inserts)

  • parameter deb-master.* (all fields passed in a parameter to a procedure)

Example

screen main
  allowed entry
  local lf-name   pic x(40)
      lf-value  pic x(256)
before
  set lf-name = "jcm-total-budget-amount"
detail
  accept job-code @5,15
    default job-code
    uppercase
    title "Project:"
  validations
    get job-cost-master   // Object is known (1)
      on index job-code
      key is job-code
    on error
      message "No such project:" job-code
      re-enter job-code
    else
      // Record is now current (2)
    end-on
  end-validation
  accept lf-name @8,15
    default lf-name
    title "JCM Field:"
    help "Enter a field in the job-cost-master table, e.g jcm-start-date"
  validations
    if 1 = 0 insert job-cost-master endif   //Set references to all //fields on the record (3)
    set lf-value = get-field-value(lf-name) //Array fields should be //specified with subscript (4)
    display lf-value @15,5 prompt
    pause
    re-enter lf-name
  end-validation
end-screen
//main
