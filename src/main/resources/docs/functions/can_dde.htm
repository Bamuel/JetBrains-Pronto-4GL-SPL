CAN-DDE()

Description
Returns a True/False result indicating whether DDE functions can be performed in the current environment
Syntax
CAN-DDE()
Category
Environment
Runtime version
5.0 and above

Notes

This function returns TRUE (non-zero) if the current environment can perform DDE functions, otherwise returns FALSE (zero) if DDE functions are not available.

This function is mainly for use with the Microsoft Windows operating system. DDE facilities are no longer supported by Microsoft and little or no documentation exists on DDE requests. Visual Basic calls to Microsoft applications can sometimes be similar in construct to DDE requests.

The use of DDE is discouraged as connection to third-party software applications and COM objects can be made using Pronto Xi OLE statements and functions.

This function has no effect in the Pronto Xi web interface.

Example

field ws-id     pic s9(4)
      ws-result pic s9(8)v99
				
procedure main
if not can-dde()
    message "DDE functions will not run in this environment"
else
    set ws-id = dde-initiate("Excel", "Sheet1")
    if ws-id <= 0
      message "Initiation Error - please start Excel first, then re-run this program"
    else
      if not dde-poke(ws-id, "r1c1", "124")
        message "poke error"
      else
        set ws-result = num(dde-request(ws-id,"R1C1"))
        if sleep(5)
        endif
        message "r1c1 is set to " ws-result
        if dde-terminate(ws-id)
        endif
        message "Finished"
      endif
    endif
endif
 
// Result: "r1c1 is set to 124.00" (assuming one can-dde)
