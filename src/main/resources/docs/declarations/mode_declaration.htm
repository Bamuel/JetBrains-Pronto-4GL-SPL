MODE declaration

This statement can be used to define screen modes.

The SEARCH, ENTRY, CORRECT, REMOVE, ENTRY-ONCE, and DUPLICATE modes are already defined by the language. However, their standard definitions can be changed using this statement.

This statement is non-procedural. It can occur anywhere in the program without affecting the program’s logic flow.

A mode defined by a MODE statement is global to the entire program. However, it cannot be referenced until after it is defined.

Syntax

mode mode-name
   [prompt [is] alpha-expression]
   [help [is] alpha-literal]
   [security [is] numeric-literal]
   [process]
   [entry|correct|remove|duplicate|entry-once|find|next-scr|prev-scr]
   [currency]
   [lock]
   [perform routine-name]
   [when condition]
   [mode-when condition]
   [colour [is] colour]
   [icon [is] icon-name]
   [always-show|always-add|no-hide]

Clauses

Clause Description

PROMPT

Defines the prompt displayed in the mode prompt line. If none is specified, mode-name is used as the prompt.

HELP

Define a help line to display for the mode.

SECURITY

Specifies the minimum security level required to access this screen mode. This applies to the user’s global security level number.

Application security does not use the global security.

ENTRY

Puts the screen into Entry mode for all fields with an associated ACCEPT statement.

CORRECT

Puts the screen into Correct mode for all fields with an associated ACCEPT statement.

REMOVE

Puts the screen into Remove mode.

This clause only has an effect when the PROCESS clause is also used.

DUPLICATE

This mode can be used to duplicate/copy the current record. It acts exactly as Correct mode during screen input, however the values are inserted into a new record, instead of updating the current record.

ENTRY-ONCE

Puts the screen into Entry mode for all fields with an associated ACCEPT statement. Single iteration only, that is, once the screen is processed, control is returned to the mode menu. Entry mode automatically positions the user for entering another record.

PROCESS

The PROCESS clause ensures the File I/O for the selected mode type is performed, for example CORRECT, REMOVE, DUPLICATE, ENTRY-ONCE, and FIND.

PROCESS CORRECT would process the fields in the correct mode and then perform an update.

If a PRIMARY object is defined for the screen, process the file with the specified mode: ENTRY, CORRECT or REMOVE.

CURRENCY

Record currency is required on the screen’s PRIMARY object, if any, for this mode to be valid.

LOCK

This causes the mode to re-read and lock the current primary object record when selected. This mode implies CURRENCY.

The record remains locked until the next operation is performed on the object.

PERFORM

Perform the specified SCREEN, MENU, or PROCEDURE whenever this mode is selected.

WHEN

The is valid for a screen only if the condition specified for the WHEN clause evaluates to TRUE.

The WHEN condition is only evaluated when the modes are initially displayed for the screen (after the BEFORE section is processed) or when the screen is redisplayed due to the issue of the REFRESH statement. Any change in the result of the WHEN condition after this point does not affect the modes available unless the screen is opened again or the REFRESH statement is run.

MODE-WHEN

This specifies a condition that determines whether or not the mode is valid for the screen. This condition is evaluated only once when the screen is opened.

If both MODE-WHEN and WHEN conditions are present, the MODE-WHEN clause indicates whether or not the mode is valid for the screen, and the WHEN condition will indicate whether or not the mode is valid for the current record.

If a MODE-WHEN condition is not specified, then the behaviour of the WHEN condition is used when a screen is opened. If there is no MODE-WHEN condition and the WHEN condition is false when the screen is opened, then the mode will not be valid for the screen unless an ALWAYS-SHOW clause is also specified.

COLOUR or COLOR

Usage examples:

colour is red on black
colour is blue
colour is red on 0

When 0 or no colour is specified, as in the third example above, the default foreground or background colour set by the user in the colour settings of the Pronto Xi Client is used.

Windows RGB colour values can be specified by using the RGB-TO-COLOUR() function.

Either the foreground colour only, or both the foreground and background colours can be specified. The colours can be specified as an expression (and can be varied at runtime). Fields storing colours must be at least PIC 9(8).

A field type of COLOUR (for example, TYPE IS COLOUR) can be used.

ICON

The icon used in screen and tree menu based buttons can be specified by using the ICON clause on both the OPTION statement and MODE declaration. Mode icons are visible upon right-mouse click.

The following predefined icon types are available:

  • FOLDER-ICON

  • ADMIN-FOLDER-ICON

  • REPORT-FOLDER-ICON

  • ADMIN-ICON

  • REPORT-ICON

  • SCREEN-ICON

  • TRANSACTION-ICON

  • PERIOD-END-ICON

  • QUESTION-ICON

  • BLANK-ICON

  • FAVOURITES-FOLDER-ICON

  • LOCKED-FOLDER-ICON

  • BLUE-FOLDER-ICON

  • RED-FOLDER-ICON

  • GREEN-FOLDER-ICON

  • ENQUIRY-FOLDER-ICON

    This icon should only be used in the Pronto Xi web interface to indicate a file upload option.

Icons are not used in the Pronto Xi web interface menu.

option "TESTING" @10,10
         icon is screen-icon

The OPTION statement allows the icon type to be specified by a numeric expression (or field name) to allow for variable options. The MODE declaration, however, does not allow expressions.

set option-title = "TESTING"
set icon-number = screen-icon
...
option option-title @10,10
         icon is icon-number

The width of the mode button can be extended up to two characters depending on the length of the mode prompt.

ALWAYS-SHOW

This clause has effect only if a WHEN clause is present for a mode.

If the WHEN clause is invalid when the modes for a screen are initially shown, the mode is not displayed by default. A mode with the ALWAYS-SHOW clause is always added to the screen regardless of the initial state of the WHEN condition.

The ALWAYS-SHOW keyword is interchangeable with NO-HIDE and ALWAYS-ADD.

Notes

  • The character required to select the MODE using the keyboard can be specified by prefixing the character with an ampersand (&) in the prompt; for example, E&xit specifies x as the selection character. To print a single ampersand, a double ampersand (&&) must be specified in the prompt.

  • If the prompt does not specify a selection character explicitly, the first printable character in the prompt is used.

  • Multiple mode definitions can be specified.

    mode mode1 correct process
mode2 entry prompt "entry"
mode3 prompt "Do stuff"
