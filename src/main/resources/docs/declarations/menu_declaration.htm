MENU declaration

Defines a menu. The routine should contain menu options (using the OPTION statement) and procedural commands to perform if that option is selected. The procedural code itself can be broken up into three command groups (BEFORE, AFTER and DETAIL).

If this is the first routine defined for the program (or its name is main), it is executed initially.

This statement automatically ends any previous SCREEN, PROCEDURE, API or MENU declaration.

The menu will loop until it is backed out of or until an EXIT statement is encountered.

The presentation of the menu (for example, tree format, popup or the traditional button format) is controlled by the TREE-MENU and NO-HIDE clauses.

Syntax

menu menu-name
   [window [@row,col to @row,col]
      [relative to @row,col]]
   [window-position [is] numeric-value]
   [local [field] field-name field-definition ...]
   [parameters [are] field-name field-definition ...]
   [returning] field-name field-definition ...]
   [button-width [is] numeric-literal]
   [stop-exit-all-key]
   [title [is] alpha-expression]
   [tree-menu] [no-hide]
   [colour [is] colour [on colour]]
   [export|user-trigger [can-override|no-override]]
   [help-context [is] help-file-expression  context-id-expression]
   [before 
      procedural-statements]
   [detail 
      procedural-statements]
   [after 
      procedural-statements]
[end-menu]

Clauses

Clause Description

WINDOW

The menu is defined as being a window over the current screen. When the menu returns, the window disappears automatically.

If no coordinates for the window are specified, the window occupies the entire screen space.

If the RELATIVE TO clause is included, the window position is relative to the specified coordinates. The row and column values can be arithmetic expressions.

Decimal addressing is not supported for data grids. The row and column coordinates for each field in the data grid must be specified as a whole number.

WINDOW-POSITION

This clause, which only takes effect if the WINDOW clause is also specified, is used to specify the initial position of the window, or to display it in full screen mode.

The allowed values are:

-1

Displays the window in full screen mode

1 to 9

Sets the initial position of the current window in the container window, where 1 = top left, 2 = top centre, and so on as shown in the following grid:

For example, to open the window at the bottom centre of the container window:

window-position is 8

LOCAL FIELD

The LOCAL FIELD declaration defines fields that are local in scope to this routine.

For more information, see LOCAL FIELD declaration.

PARAMETERS

Allows the definition of parameters that are passed to the routine.

For more information, see PARAMETER declaration.

RETURNING

Allows the definition of parameters that are returned to the calling routine.

For more information, see RETURNING declaration.

BUTTON-WIDTH

This clause allows the width of the buttons used when displaying the prompts for the OPTIONs to be controlled (in versions that support button drawing). When running on a version of Pronto Xi that supports buttons, all buttons for the MENU are the same size by default. The size being that of the largest prompt for any of the OPTIONs in the menu (regardless of its row/column position).

A positive value indicates that all buttons must be at least this size (in characters). A value of zero indicates that the button sizes are calculated using the default method. A negative value indicates that all buttons are the size of their prompt, but the text is centred in the button (as for a MODE).

menu test-menu
  button-width is 35

STOP-EXIT-ALL-KEY

When F8 is pressed, the runtime automatically backs out all programs being run by the user.

It accomplishes this by flagging that all requests for keyboard input are to be treated as though the Esc key had been pressed, to try and allow the programs to back out gracefully. All screen output is stopped while in this mode.

The EXIT-ALL operation is cancelled if the program executes a BACK-TO-DETAIL statement, as the intention of this statement is to stop users from being able to exit the screen by pressing Esc.

The STOP-EXIT-ALL-KEY clause was added to the SCREEN, MENU and PROCEDURE declarations to also stop the EXIT-ALL operation. The automatic backout is cancelled when a routine with this clause specified is encountered.

When this clause is included in a routine, the EXIT-ALL operation is not stopped if the user presses the EXIT-ALL key while in the routine. It only takes effect if the EXIT-ALL key is pressed in a child routine.

Any code that loops to stop a user from being able to exit by pressing Esc (other than by using the BACK-TO-DETAIL statement) can cause the application to hang if the EXIT-ALL key is pressed. Any routines that contain this type of code should either be rewritten or specify the STOP-EXIT-ALL-KEY clause to avoid this problem.

As a fail-safe, a 10 second limit is placed on the EXIT-ALL operation. If the EXIT-ALL is still in effect after 10 seconds, it is automatically cancelled.

This should not be relied upon, as this timeout is not available on all systems.

TITLE

The TITLE clause is available for any MENU/SCREEN/PROCEDURE with a window. This will set the title on the title bar of the window. It is processed once only upon entry to the routine.

TREE-MENU

NO-HIDE

The TREE-MENU clause on a MENU can be used to indicate to the runtime that this menu should be displayed as a tree style menu instead of a popup menu.

In the Pronto Xi web interface, the tree menu refers to the left hand side standard menu.

If a valid tree menu is active (without child screens open), the menu will form part of this tree menu. Otherwise, a new tree menu is created.

If the MENU contains the NO-HIDE clause the TREE-MENU clause has no effect. If the MENU routine does not define its own window, a NO-HIDE clause on the routine that defined the window will also override the TREE-MENU clause.

Multiple tree menus are supported.

A tree view menu system is provided for navigating through the application’s menu instead of the traditional button based menus.

A menu displayed while there are no visible windows (excluding the tree view) is shown in the main tree view menu by default.

Menus are defined in a program in the traditional way. A menu shown as a tree view (or popup) ignores any DISPLAY statements that are present.

When running on a character based device, the traditional menu is used.

Any menu opened while there are visible windows (excluding the tree view) is displayed as a popup menu by default.

The NO-HIDE option on the MENU causes the menu to be displayed in the traditional way instead of using a popup (or tree view).

A popup menu is restricted to a maximum of 25 menu items.

The position of the popup is based on the top/left coordinates of the last input field or menu option used.

Any popup menu should exit once the option is performed.

The CONFIRMED section of the CONFIRM should finish with an unconditional EXIT.

The positioning of the popup menu after the user selects an option is not guaranteed.

COLOUR or COLOR

The COLOUR clause is valid for a SCREEN/MENU/PROCEDURE to specify the default window colours. If only a single colour is specified, it is assumed to be the background colour.

Usage example:

colour is red on 0

When 0 or no colour is specified, the default foreground or background colour set by the user in the colour settings of the Pronto Xi Client is used.

Windows RGB colour values can be specified by using the RGB-TO-COLOUR() function.

This clause is only valid if the WINDOW clause is also specified (with the exception of the main window).

EXPORT

Indicates that the routine is an entry point to an operations table when used as a component library.

For more information, see EXPORT clause.

USER-TRIGGER

This clause is similar to the EXPORT clause described above, but also allows the routine to be called as a component library trigger by Screen Customiser.

CAN-OVERRIDE

NO-OVERRIDE

These clauses specify whether the routine is allowed to be overridden by custom component versions. If neither clause is included, CAN-OVERRIDE is assumed.

HELP-CONTEXT

Specifies a context-sensitive help topic to display when the user presses F1 while this menu is being executed.

help-file-expression is an alpha expression or literal that specifies the name of the help file. context-id-expression is a numeric expression or literal that specifies the context ID of the topic in that help file that is to display.

This setting will override the help context information stored in the System Modules & Functions (SYS M412) function for the menu function that was used to access this menu, or a help context defined using the SET-HELP-CONTEXT function.

This feature requires runtime version 7.1 or higher and a .chm, .hlp or .htm format context-sensitive help system that has been generated using a suitable help authoring tool.

BEFORE

Specifies the BEFORE command group. This should contain any set up code for the menu (for example, headings). This is performed once at the start of the menu.

DETAIL

This contains the iterative code for the menu. This code is performed for each selection from the menu. This command group should contain OPTION statements for every menu entry.

AFTER

Specifies the AFTER command group. This should contain any finalisation code for the menu. This is performed once when closing the menu.

Notes

  • There is a special case where the name of the menu is called main. In this circumstance the procedure is the first performed for the program, regardless of its location in the program.

  • Parameters for a routine can be passed and returned as a complete row/record for an object. For more information, see PARAMETER declaration.
