PROCEDURE declaration

This defines a standard procedure containing a list of Pronto 4GL statements which are executed sequentially in top-down order.

A procedure is called in the same way as a screen, menu or APIAn API specifies how software components should interact with each other. APIs are commonly used to specify the remote calls exposed to an external application, and to ease the work of developing graphical user interface (GUI) components. In a web-services architecture, Web APIs come in a variety of formats with the most common being REST and SOAP. Pronto Connect has adopted a RESTful approach and offers standard out-of-the-box APIs as well as a development framework for the creation of custom APIs.. If this is the first procedure defined for the program, or the name of the procedure is main, this is executed initially.

This statement automatically ends any previous SCREEN, PROCEDURE, API or MENU declaration.

Unlike menus and screens, procedures do not loop automatically; they return once the end of the procedure is encountered.

A procedure can take input from the screen through one or more ACCEPT statements, if required. Each ACCEPT statement is processed in turn, and the user has the opportunityIn CRM, an opportunity is an account that has qualified for potential sale. This type of account usually leads in the direction of a quote being submitted. Opportunity Management helps Sales teams manage opportunities and identify their position within an opportunity so they can plan the best strategy for closing the deal. to arrow back to a previous ACCEPT. Generally, however, for re-iterative screen input, a screen declaration should be used.

The EXPORT, USER-TRIGGERA trigger is a routine that is automatically run (triggered) whenever a specific condition occurs., CAN-OVERRIDE and NO-OVERRIDE clauses can be used on screens, menus, procedures or API declarations.

Syntax

procedure procedure-name
   [window [@row,col to @row,col]
      [relative to @row,col]]
   [no-hide]
   [window-position [is] numeric-value]
   [colour [is] colour [on colour]]
   [local field-name field-definition ...]
   [parameters [are] field-name field-definition ...]
   [returning field-name field-definition ...]
   [stop-exit-all-key]
   [title expression]
   [export|user-trigger [can-override|no-override]]
   [auto-transaction]
   [help-context [is] help-file-expression  context-id-expression]
   procedural-statements
[end-procedure]
