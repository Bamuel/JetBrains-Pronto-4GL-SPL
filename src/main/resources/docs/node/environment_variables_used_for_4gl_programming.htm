Environment variables used for 4GL programming

Certain environment variables can be set to change or specify the way in which Pronto Xi performs certain functions.

The following environment variables are recognised by Pronto Xi.

4GL Environment Variables
Environment variable Description

PRONTO

This environment variable must be set.

Specifies the Pronto Xi home directory.

This is the parent directory of the subdirectories lib and bin that contain the Pronto Xi system files and programs.

BMS

Specify the parent directory for the Pronto Xi applications software.

Programs for each Pronto Xi module are contained in one subdirectory of this directory.

CUS

Specify the directory where customer specific applications are installed.

EDIT

Specify the name of the default text editor.

NETPRONTO

For Microsoft Windows implementations only.

The full network path to the Pronto Xi directory on the server machine.

PATH

Specify the search path list for external (non-Pronto Xi) commands executed.

PROCMPCT

Enable codified text translations when using procmp to compile a program.

PROCMPINC

Specify the path to the directory in which the include files are located when using procmp to compile a program.

PROCURRENCY

Set the default currency symbol (for example, $) to use.

PRODATABASE

For RDBMS implementations only.

Specify the name of the database instance containing the Pronto Xi data.

If not set, the default database instance for the machine is used.

PRODATEFMT

Specify the required date format (day, month, and year ordering) to use.

1

DD/MM/YY Ordering

2

MM/DD/YY Ordering

3

YY/MM/DD Ordering (not currently supported)

PRODECPT

Specify the default decimal point character to use.

The allowed values are ".", or "," (include the quotes when setting the environment variable).

If the U_PRODECPT override program exists in the override program table, and the entered value in the PRODEDPT environment variable is a comma, the values are exported as they appear in the Pronto Xi screen.

PRODEFDATAGRID

This environment variable is intended specifically for Software Developers and Testers.

This environment variable allows Developers and Testers to run applications with their default data grid settings, eliminating confusion caused by user changes to them.

When this environment variable is set to a non-null value, all columns in the default view of a data grid are displayed at their default widths and any stored user modifications are ignored.

PRODICT

Specify the standard dictionary file to use for this session.

PRODISTDICT

Specify the distributor-supplied dictionary to be used in addition to the standard and user dictionaries.

PROLANGDIR

Specify the name of the database containing the language translation tables used by procmpnls to generate the NLS files for each program.

The NLS file contains the language translation information for a program.

PROLANGUAGE

Specify the foreign language to be used.

For example, US-English, French, Mexican, or Spanish.

PROLOGINTLOCKS

Write log interactive locks to a file for analysis purposes only.

PROLOGLOSTLOCKS

Write a log of lost locks caused by the SERIAL 4GL statement to a file for analysis purposes only.

PROMONTHS

Comma-separated list of 12 month names to override the default English names.

PROPATH

Specify a list of directories to search for SPL programs and SQL enquiries.

PROPRINTERS

Specify the set of printers that are available to the user.

PRORUNSTATS

Specify the path and file name that trace statistics are to be written to for the Realtime Trace Control (SYS M025) function.

PROSINGLEKEY

Override system-wide setting for single keystroke menu/mode selection.

The value Y or y turns on single keystroke menu selection. The value N or n turns it off.

PROTERM

Specify the terminal type on which Pronto Xi is being run.

PROTIMEOUT

Automatically log users out of Pronto Xi after a specified number of minutes of inactivity.

PROUSRDICT

Specify the custom dictionary to be used in addition to the standard dictionary.

PROWEEKDAYS

Comma-separated list of 7 day names to override the default English names.

PROXMLDETAILBG

Specify the default background colour for the report detail/body when generating an XML/XSL report (when no customised style sheet is being used).

PROXMLDETAILFG

Specify the default foreground colour for the report detail/body when generating an XML/XSL report (when no customised style sheet is being used).

For more information, see PROXMLDETAILFG environment variable.

PROXMLDISPLAYSIZE

Specify the default display font size when generating an XML report (when no customised style sheet is being used).

PROXMLENCODING

Specify the XML character set code used when generating an XML report; for example, ISO-8859-1 or GB2312.

PROXMLHEADERBG

Specify the default background colour for the report page header when generating an XML/XSL report (when no customised style sheet is being used).

PROXMLHEADERFG

Specify the default foreground colour for the report page header when generating an XML/XSL report (when no customised style sheet is being used).

PROXMLPRINTSIZE

Specify the default print font size to use when generating an XML report (when no customised style sheet is being used).

PROXMLREPORTS

Specify the report format, and how the report is to be viewed.

PROYEAROFFSET

Specify the number of years to offset from the system date.

This is intended in countries whose years are not based on the Gregorian calendar.

REPORTDIR

Specify the directory in which to place all report (spool) files generated.

If this is not defined all reports are spooled into your current directory (usually the data directory).

SHELL

Specify the default shell to run when a shell is called.

This is applicable only to UNIX systems.

TERM

Specify the terminal type name as defined in the terminfo database (for example, VT100).

If this is defined it overrides any default terminal type defined for your terminal in Pronto Xi.

This applies only to UNIX/Linux based machines.

TMPDIR

Specify the directory in which to place all temporary files that are generated.

If this is not defined, the standard temporary files directory on your system is used.

TZ

Specify the time zone environment.

This is used for setting the time zone. It is used by various time functions to calculate times relative to Coordinated Universal Time (UTC), formerly known as Greenwich Mean Time (GMT). The format is specified by the operating system. This environment variable will affect the TOD(), TODAY(), and TIME-ZONE() functions.

Path lists as required by PROPATH and PATH are a set of one or more directory names (usually specified with their full path), each separated by a colon (:). The current directory is not searched unless it is specified in the list or a colon appears first in the list. Directories are searched in the order that they appear in the list.

On Microsoft Windows systems, the separator for a path list is a semi-colon (;) since colons are used to specify a device.
