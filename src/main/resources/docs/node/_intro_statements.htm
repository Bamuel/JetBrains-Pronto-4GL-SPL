Statements

The Pronto RAD programming language supports the following statements:

Statements by category

Program execution

  • SPL / CALL

    Executes the specified 4GL program.

  • CALL-URL

    Calls a specified URL.

  • COMMAND

    Executes the specified operating system command or program.

Responsive screen programming

  • DROP-DOWN

    Defines a drop-down list in a responsive screen.

  • FIELD-GROUP

    Groups a series of fields or other displays together in a responsive screen.

  • SCREEN-GROUP

    Defines a screen group in a responsive screen.

  • SCREEN-SECTION

    Defines a screen section in a responsive screen.

Program control

  • ABORT

    Displays a message and aborts the program after message acknowledgement.

  • ACKNOWLEDGE

    Displays a message and resumes program execution after message acknowledgement.

  • BACK-TO-DETAIL

    Returns execution to the mode selection of a screen, or the option selection of a menu.

  • BREAK

    Exits the current loop and resumes execution at the first statement after the loop

  • CONFIRM

    Prompts for user confirmation in a screen, procedure or menu.

  • CONTINUE

    Ignores all further statements in the current loop and resumes execution at the next iteration.

  • CONTINUE-ENTRY

    Ignores all further statements in the DETAIL section of a screen and resumes at the start of the DETAIL section.

  • DO

    Performs the specified SCREEN, PROCEDURE, API or MENU defined in the current program.

  • EXIT

    Exits the current SCREEN, PROCEDURE, API or MENU.

  • FOR

    Initialises the value of a specified field and repeats the enclosed procedural statements until the field value exceeds the specified TO value.

  • IF

    Conditionally performs procedural statements

  • MESSAGE

    Displays an information message on the screen.

  • MESSAGE-BOX

    Displays a configurable multi-line message box.

  • PAUSE

    Displays an OK button in the bottom-right of the current window and suspends processing until the button is pressed.

  • RE-ENTER

    Transfers control back to the previous ACCEPT statement in the current screen.

  • REPEAT

    Executes a first section of procedural code once initially, then the first and second sections of procedural code conditionally until a specified condition is met.

  • SWITCH

    Performs different sections of procedural code depending on the result of an expression.

  • WHILE

    Repeatedly performs the procedural statements while a specified condition evaluates to TRUE.

  • WEB-CLIENT-LOCAL-AGENT

    Controls when Local Agent is to be used when interacting with the Pronto Xi web interface.

Screen elements

  • ACCEPT

    Displays a field on the current screen for entry/correction of data.

  • BOX

    Displays a box or horizontal/vertical line on the screen.

  • CHECK-BOX

    Displays a check box control for a field with a logical TRUE/FALSE (on/off) value.

  • DISPLAY

    Displays the result of an expression on the screen.

  • OPTION

    Displays an entry in a menu, or a button on the screen.

  • RADIO-BUTTON

    Displays a radio button group control for a field with defined values.

Screen display

  • CLEAR

    Clears all or part of the screen.

  • REFRESH

    Refreshes the current screen or menu, or a specified field.

Record/field operations

  • DELETE

    Deletes the current record of the specified object.

  • EXTRACT

    Performs conditional looping while sequentially reading records from the specified object.

  • GET

    Reads a record from the specified object.

  • INITIALISE

    Resets the values of the fields in the specified object to their default null value.

  • INSERT

    Inserts a new record into the specified object.

  • LOCK-METHOD

    Specifies how the runtime should handle locked records.

  • POP

    Restores the field values previously saved using the PUSH statement.

  • POSITION

    Positions the object at a particular record.

  • PUSH

    Temporarily saves the values of all fields in the specified object.

  • QUERY

    Executes the specified SQL query and displays the results on screen or spools them to a report file.

  • RE-SELECT

    Restarts the SELECT operation of a screen.

  • RESTORE

    Restores the field values previously saved using the SAVE statement, or initialises the fields of the specified object.

  • SAVE

    Temporarily saves the current values of all fields in the specified object.

  • SELECT

    Reads and processes the records of the specified object or objects.

  • SERIAL

    Reads the specified object and serially increments the value of a specified field.

  • SET

    Assigns a value to a field in a specified object, or declared in a FIELD statement.

  • SQL-DELETE

    Deletes one or more rows of a table, based on specified conditions, without executing any 4GL triggers or auditing.

  • SQL-UPDATE

    Updates one or more rows of a table, based on specified conditions, without executing any 4GL triggers or auditing.

  • STRING

    Manipulates the contents of an alphanumeric field of type ALPHA or STRING.

  • UNLOCK

    Unlocks the current record for the specified object, or unlocks all locked records for all objects.

  • UPDATE

    Updates the values of the current record of the specified object.

Transaction processing

  • BEGIN WORK

    Marks the beginning of a set of operations that are subject to transaction control.

  • COMMIT WORK

    Commits the operations since transaction control was enabled.

  • ROLLBACK WORK

    Rolls back the operations since transaction control was enabled.

  • TRANSACTION

    Enables transaction control, or commits or rolls back the operations since transaction control was enabled.

Report generation

  • NEED

    Forces a new report page if the number of lines left in the current page is less than the specified amount.

  • PAGE

    Forces a new page for the report.

  • PRINT

    Prints the result of an expression on the report.

  • REPORT

    Starts/finishes a report

  • REPORT SECTION

    Marks the start or finish of an XML section in the report.

  • SKIP

    Skips to a specified line number and forces a new page if there are insufficient lines remaining on the page.

File handling

  • CLOSE

    Closes the physical file associated with an object.

  • OPEN

    Opens the physical file associated with an object.

Miscellaneous

  • AUDIT

    Activates/deactivates auditing of changes to specified objects.

  • DISABLE-ALL-TRIGGERS

    Disables all data trigger processing in the current program and any attached component libraries.

  • LINK

    Attaches a component library to a program so that the associated routines can be referenced.

  • STATEMENT-GROUP

    Marks a group of statements in the 4GL program that must be kept together.

  • VERSION-NUMBER

    Sets the version number of the current 4GL program.
