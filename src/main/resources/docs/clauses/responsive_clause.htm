RESPONSIVE clause

The RESPONSIVE clause specifies that a Pronto Xi web interface screen is responsive. Within the SCREEN declaration, the RESPONSIVE clause must be used to indicate that the screen is responsive, otherwise it is a fixed position (non-responsive) screen.

Syntax

responsive

Within the SCREEN declaration, the RESPONSIVE clause must be used to indicate that a screen is responsive, otherwise it is a fixed position screen.

When a screen is declared as responsive, then every command that displays something on the screen must be contained within a SCREEN-GROUP statement.

For example, any ACCEPT, DISPLAY, BOX, CHECK-BOX, RADIO-BUTTON, OPTION, and so on, must be contained within a SCREEN-GROUP.

Responsive screens are automatically FORM-ENTRY and FORM-ENTRY does not have to be separately specified.

The RESPONSIVE clause effectively replaces the WINDOW clause in a screen because a responsive screen will adjust dynamically to the available space.

A responsive screen does not require a WINDOW clause because it relates to a screen of variable boundaries. This includes screens specified to be a dialog, the boundaries of a dialog will be automatically calculated to suit the information specified for the dialog without requiring a WINDOW clause.

In a responsive screen, the number passed by WINDOW-POSITION determines the relative width of the dialog in relation to the screen.

Default coordinates for responsive screens

  • The ACCEPT, CHECK-BOX, RADIO_BUTTON, and DISPLAY statements do not need to specify @row,column coordinates if they are part of a responsive screen.

  • When part of a SCREEN-GROUP, the item will be placed below the last item for the group within the same column.

  • When part of a FIELD-GROUP, the item will be placed to the right of the previous item in the group within the same row.

  • The NEXT COLUMN and NEXT RIGHT clauses will override the default positioning to place the item to the right of the previous item in the group within the same row.

  • The NEXT ROW clause will override the default positioning to place the item in the first column of the row below the previous item in the group.

Example

screen journals
   responsive
       title 'Journals'
