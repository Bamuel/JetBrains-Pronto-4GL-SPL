DROP-DOWN statement

A drop-down is a graphical control element that allows the user to choose one value from a list in a responsive screen.

When a drop-down is inactive, it displays a single value.

When activated, it displays (drops down) a list of values, from which the user can select one.

When using a drop-down, the entries in the drop-down are automatically validated, so you do not need to use a separate validation to confirm that the entry exists in the database.

Requesting a drop-down list for a field that is not the current field is supported.

Syntax

drop-down
   [dynamic]
   [optional
   [edit]
   [concat-title]
   [width [is] expression]
   [[values <value-expression>
   [TITLE [is] <expression>] [ICON [is] <expression] [WHEN <condition> [SHOW-VALUE]]]
   [select <value>, [<title>, [<icon-number>, [<allow-selection-boolean>]]]
   <select-clauses>]
end-drop-down

Clauses

Clause Description

DYNAMIC

By default, the drop-down list is considered to be static for the screen.

Add this clause to make the drop-down list dynamic, that is, allow the list content to vary, depending on the selected record.

Every time that the data record changes based on a Find, Up, Down, Pg Up, Pg Down, and so on, the drop-down list is redetermined.

Do not make a drop-down dynamic unless necessary.

The DYNAMIC clause is applicable only when the drop-down is subject to conditional values.

OPTIONAL

Allows optionally populated fields to display a blank in a drop-down. This is useful when you want to use drop-downs for display purposes only, and not accepting data.

The OPTIONAL clause adds a blank record to the end of the drop-down list. This allows for the field being populated by the drop-down to remain blank.

This should not be used when the field needs to be validated against an entry in a database table.

When the field needs to be validated against a database table, and blank is a valid value, then a blank record must exist in the database table, and the OPTIONAL clause should not be used.

EDIT

Specify that the DROP-DOWN is only a suggestion list. That is, any value can be entered.

Otherwise the value must be in the list. This is a combo box.

drop-down concat-title width 45
   edit
   select sys-tbl-code sys-description from system-table
end-drop-down

CONCAT-TITLE

The CONCAT-TITLE clause can control the drop-down description.

The drop-down values / select include a code and a description of that code.

Without the CONCAT-TITLE clause, the drop-down displays only the description.

With the CONCAT-TITLE clause, the drop-down displays the code, a hyphen, and the description.

Most drop-downs should include this clause.

drop-down concat-title
   'A' title 'Alpha Validated'
   'C' title 'Comment Non Validated'
   'D' title 'Date'
   'N' title 'Number'
   'V' title 'Alpha Non Validated'
   'X' title 'Existence Only No Data'
end-drop-down

WIDTH

Specify the width (in characters) of the drop-down list that is shown on screen.

This might be needed when titles / descriptions are specified to be shown instead of values.

If not specified then the Pronto Xi web interface must determine the appropriate width.

The width for a drop-down is specified in characters, which is different from the width for a screen group. In a screen group, the width is a proportion of the horizontal space of the screen.

VALUES | SELECT

There are two types of drop-downs:

  • VALUES

    A hard-coded list of code descriptions.

  • SELECT

    A list of codes and descriptions returned from a SELECT statement.

The SHOW-VALUE clause can be specified for an entry in the VALUES list. This indicates that if the WHEN clause for the value is FALSE, that the value is still shown in the list (but is unavailable for selection).

A SELECT statement for a drop-down requires up to four returning values; the first two are mandatory. The fourth field is optional. If this field is specified, it is used to indicate if the entry is valid for selection. A zero value indicates that it will be unavailable for selection.

The four returning values are:

  • code

  • description

  • icon

  • show value boolean

Hard coded list (VALUES):

drop-down
   width 30
   concat-title
   'W' title "Won"
   'L' title "Lost" 
   'E' title 'Entered'
end-drop-down

SELECT statement:

accept stk-abc-class
   title "ABC Class"
   default stk-abc-class
   uppercase
   drop-down
      width 30
      concat-title
      select sys-tbl-code sys-description from system-table where sys-tbl-type = "IA"   
   end-drop-down

Conditional drop-down values (WHEN clause)

In a hard coded drop-down, use of the [when condition],[show-value] elements of the command can control what the user sees in different circumstances.

In a drop-down based on a SELECT statement, the control of conditional values is within the SELECT statement itself.

That is, the WHEN clause on the SELECT statement restricts the returned values, or the 4th returning value of the SELECT statement controls whether or not the value can be selected.
