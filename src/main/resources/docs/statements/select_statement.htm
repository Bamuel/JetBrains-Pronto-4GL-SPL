SELECT statement

Reads and processes the records of the specified object or objects.

The SELECT statement is a procedural looping structure that reads rows from the specified table or tables. Each returned row forms one iteration of the loop.

Syntax

select
   select-clauses
   [on error [error-number ...]  procedural-statements end-on] 
   [before [field-list]
      procedural-statements]
   [detail
      procedural-statements]
   [after [field-list]
      procedural-statements]
end-select

Clauses

Clause Description

select-clauses

For details, see SELECT clause.

ON ERROR

Specifies procedural statements to perform if the SELECT fails. A list of specific error numbers to which the ON ERROR clause applies can optionally be specified.

This clause must be located immediately before the first BEFORE, DETAIL or AFTER clause, and cannot contain an ELSE clause.

You cannot trap EENDFILE as this is not considered an error for the loop.

This clause is supported for runtime versions 7.1 and above.

BEFORE

Specifies procedural code that is to be performed before the DETAIL section when one or more of the specified fields changes its value. Current field values are those of the new record. Multiple BEFORE sections can be used.

A BEFORE section specified without a field list is performed once at the start of the SELECT loop if the SELECT loop processes records. It is not performed if no records are processed.

DETAIL

Specifies procedural code that is to be performed once for every selected row.

Multiple DETAIL sections are not allowed.

AFTER

Specifies procedural code that is to be performed after the DETAIL section when one or more of the specified fields changes its value. Current field values are those of the previous row. Multiple AFTER sections can be used.

An AFTER section specified without a field list is performed once at the end of the SELECT loop if the SELECT loop processes rows. It is not performed if no rows are processed.

Notes

  • The SELECT loop can abort with a fatal runtime error if the objects in the FROM list are referenced in another file I/O operation that would affect the current position of the object, as the integrity of the SELECT loop cannot be guaranteed.

  • The UPDATE and DELETE statements can be used on the row returned from a single table SELECT as long as the FOR UPDATE clause is also used.

Example

The following code demonstrates how to read backwards through a table. The WHERE clause defines the starting position, the IF statement in the DETAIL clause defines a finish when condition, and the DESC option in the ORDER BY clause sorts the fields in descending order.

set lf-save-flag = sol-print-line
// first read backwards to a non DN, so we will edit all the notes!
//
// we are current on a s/o line so start reading backwards from here
select
    *
from like-sales-order-line
  order by so-order-no desc so-bo-suffix desc sol-line-seq desc
where (so-order-no = :lf-save-order-no
  and so-bo-suffix = :lf-save-suffix)
  and sol-line-seq < :lf-save-start-line
detail
  if sol-line-type != 'DN'          // no longer a note line
  or sol-chg-type in { 'T' 'D' }    // is part of kit
  or sol-print-line != lf-save-flag // diff print flag
  then
        //if the previous line is not a DN or it is part of //a kit or the print flag is different or the note //type is different, break out of the select.
    break
  endif
  set lf-save-start-line = sol-line-seq
end-select
