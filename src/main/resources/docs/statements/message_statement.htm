MESSAGE statement

Displays an information message on the screen.

If multiple expressions are specified, they are concatenated and displayed as one message.

Syntax

message expression [pic [is] picture-string]
   [expression [pic[ture] [is] expression]]
      .
      .
      .
   [no-bell]

Clauses

Clause Description

PICTURE or PIC

Allows the default format to be overridden in which the preceding expression is displayed. The picture can be an expression that is evaluated at runtime.

NO-BELL

By default, the terminalâ€™s bell will sound when a message is displayed. This clause stops the bell from sounding.

Notes

  • Messages are displayed in a popup message box window with an OK button. The user is required to press the OK button to acknowledge the message before execution of the program will continue.

  • Messages with the NO-BELL clause are displayed on the status line at the bottom of the controlling window frame. A MESSAGE statement with a NO-BELL clause that is immediately followed by the PAUSE statement is also displayed in a message box/window.

  • If the PAUSE statement immediately follows a MESSAGE statement without the NO-BELL clause, the PAUSE is ignored (to avoid an additional unwanted user interaction).

  • When running on a dumb character based terminal, the MESSAGE statement displays a message at the bottom of the screen and continue executing the program. It does not imply a pause. If a subsequent MESSAGE statement is encountered before the program requires input from the user, the system displays PRESS ENTER in the bottom left hand corner and wait for the user to do so before displaying the subsequent message. This ensures that the user sees prior messages.
