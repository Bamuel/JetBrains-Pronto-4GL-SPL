REPORT statement

Starts/finishes a report.

Syntax

report [report-title]
   [file [is] file-name]
   [direct [to] printer-name]
   [name [is] report-name]
   [header [is] procedure-name]
   [footer [is] procedure-name]
   [width [is] expression]
   [depth [is] expression]
   [length [is] expression]
   [margin [is] expression]
   [default font [is] expression]
   [full-page]
   [form type [is] alpha-literal]
   [no-xml]
   [spool-only]
   [full-xml]
   [no-message]
   [on error
      procedural-statements
    [else
      procedural-statements]
    end-on]
            
report finished [on report-name]

Clauses

Clause Description

report-title

An alphanumeric expression that identifies the title of the report.

FILE

A report by default is written to a file with an automatically generated (random) file name. If this clause is included, the report is written to a file of the specified name. file-name can be any valid alphanumeric expression.

DIRECT

By default, reports are written to a file for later printing. If this clause is included, the report is spooled directly to the specified printer. printer-name can be any valid alphanumeric expression.

Only one DIRECT report can be active at any one time. If multiple DIRECT reports are required simultaneously, finish the active report before starting the next.

NAME

Specifies an identifier name associated with this report.

REPORT names cannot be duplicated, that is, the report name must be unique to that report. This name can be referred to by any of the valid report operations.

HEADER

Defines a procedure to perform at the start of every new page.

The header is not automatically performed at the start of the report. If it is required there it must be forced using the PAGE statement.

FOOTER

Defines a procedure to perform at the end of every page.

The footer is not automatically performed at the end of the report. If it is required there it must be forced using a PAGE statement.

WIDTH

Defines the width of the report.

If not specified the width is assumed to be 132 characters.

DEPTH

The number of the last line to be used for normal printing. When this line is reached, the page footer (if defined) is automatically performed and a new page generated.

The depth value cannot exceed the length of the report. If not specified, a default depth of 6 lines less than the LENGTH is used. If there is no DEPTH or LENGTH specified, a default depth of 60 is used.

LENGTH

Defines the page/form length of the report.

The printer which the report is sent to will have its form length automatically set to the amount specified, if the printer has this capability.

MARGIN

Defines the margin for the report.

If not specified the margin is assumed to be zero.

DEFAULT FONT

The default font to use when no other is specified in a PRINT statement.

Font zero is assumed to be the default font unless this clause is used. Fonts can be numbered from 0 to 15 inclusive.

FULL-PAGE

The FULL-PAGE clause allows the full page of the report to be addressable when printing.

Line and column coordinates can be specified in the PRINT statement using the @ clause similar to the DISPLAY statement.

FORM TYPE

The name to be given to the form type for this report.

If FULL-XML is used, any customised stylesheet or customised CSS file for this report must be named based on this form type.

For more information on customised CSS and where the form name specified by the alpha-literal will be searched for, see Customised CSS files and stylesheets.

NO-XML

The NO-XML clause can be used to disable XML/XSL reporting for a report.

The report is then generated in the traditional (older style) report format (that is, #P format).

SPOOL-ONLY

The SPOOL-ONLY clause on a REPORT statement specifies that the XML viewer should not be automatically opened to view the report.

FULL-XML

The FULL-XML clause on a REPORT statement indicates that the report is to be generated as a fully compliant XML/XSL report. This requires all XML sections to be identified in the report by use of the REPORT SECTION statement and the SECTION clause on the PRINT statement.

NO-MESSAGE

Normally when a spool file is created a message indicating this fact is displayed on the screen. This clause, if used, will inhibit the displaying of this message.

ON ERROR

Perform these statements if the report specified could not be generated.

If the ELSE clause is included, the procedural statements following the ELSE are performed if the operation was successful.

The FILE-STATUS() function can be used to return the number of the error.

FINISHED

Flag that the current report is finished.

If this clause is not included, the current report is automatically finished when the program terminates.

ON

ON report-name, if used, indicates that this print operation is only for the specified report.

Notes

  • Multiple unnamed reports can be defined in a program, but only one unnamed report can be active (current) at any one time since a PRINT, PAGE, NEED or SKIP statement without the ON clause is not associated with a particular report.

  • When generating multiple unnamed reports at the same time, you can switch between reports (that is, change the current unnamed report) by re-executing the REPORT statement that created the required report. This does not apply for reports with the NAME clause specified.

Customised CSS files and stylesheets

When generating a new XML report, a customised stylesheet, or customised cascading stylesheet (CSS) for the report is searched for, based on the name of the specified form type. The name of the customised stylesheet searched for is that of the form type with an .xsl extension. The customised CSS file name is that of the form type with a .css extension.

Customised stylesheets are located in a custom_xsl directory. Customised CSS files are located in a custom_css directory. These directories are searched for in the following order:

  1. In the current data directory.

  2. In the current report directory (if different from data directory).

  3. In the $PRONTO directory.

    This is $NETPRONTO for a Microsoft Windows operating system.

  4. In the $BMS directory.

If a customised stylesheet is found for the form type, a customised CSS file is not searched for.

If no form type is specified for a report, a customised CSS file is still searched for. If the FULL-XML clause was specified, a CSS file called full-xml.css is searched for, otherwise a CSS file called basic-xml.css is searched for.
