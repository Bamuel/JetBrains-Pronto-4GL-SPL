CALL-URL statement

Calls a specified URL.

The actions performed and what data is sent and/or received is dependent on the specified parameters.

Syntax

call-url url-expr 
    [http-headers [are] headers-expr|file [is] file-expr]
    [http-body [is] body-expr|file [is] file-expr]
    [http-method [is] method-expr]
    [http-proxy [is] proxy-expr]
    [http-auth-any]
    [timeout [is] numeric-expr]
    [form name [is] name]
        [data [is] text-data]|[file [is] file-path]
        [type [is] data-type-name]
    [ca-certificate [is] file-expr]
    [ssl-certificate [is] file-expr]
    [ssl-key [is] file-expr [ssl_key_password [is] alpha-expr]]
    [allow-redirection]
    [sftp-command [is] command-string]
    [sftp-post-command [is] command-string]
    [sftp-create-dirs]
    [url-user [is] user-name [password]
    [url-login-options options-string]
    [returning
        [http-headers [in] headers-field|file [is] file-expr]
        [http-body [is] body-field|file [is] file-expr]]
    [on error [error-number ...]
       procedural-statements
     [else
       procedural-statements]
     end-on]

Clauses

Clause Description

HTTP-HEADERS

HTTP-BODY

RETURNING

Any headers or body sent can be specified as an alpha expression or passed in a file. A BLOB field can be used in place of an alpha expression.

Any headers or body returned can be stored in a field or written to a file. A BLOB field can be used to store the returned data.

Multiple HTTP-HEADER clauses can be specified. Each will be added to the header and separated by CR/LF.

HTTP-METHOD

The following standard HTTP methods can be specified:

  • get

    Requests a representation of a resource.

  • head

    Requests only the header of a resource.

  • put

    Uploads a representation of a resource.

  • post

    Submits data to a resource for processing.

  • upload

    Transfer file to the remote system. This is used with the sftp protocol.

The following additional custom methods can be used:

  • socket

    Allows custom communications. It will connect to the host specified by the URL. If a body has been specified, the body is sent. If a returning body is specified, the runtime waits for a response then a single response packet is written to the body field or file. A standard HTTP URL format should be used even though HTTP is not the protocol used. "http:" and "https:" are valid.

  • close

    Close any persistent connections. For improved performance, connections can remain open or cached.

Any unknown methods are passed through as a 'custom'.

HTTP-PROXY

The HTTP-PROXY expression can be specified in the following format:

http://username:password@host:port

All parts other than the host name are optional. If HTTP-PROXY is not specified, however, the environment http_proxy is set, the proxy information is used.

HTTP-AUTH-ANY

Specifies that the call should query the site and pick the best HTTP authentication method supported.

This might result in an extra network round trip.

TIMEOUT

The timeout value needs to be specified in seconds.

FORM

Specifies form data to be posted as the content/body of the call.

Multiple FORM clauses can be specified for a multi-part form.

NAME is the name of the form item.

DATA specifies the content as a text field.

FILE specifies a the path of a text file that contains the content.

TYPE is optional and specifies the type of form data.

If form data is specified then a HTTP-BODY cannot also be specified.

CA-CERTIFICATE

Specifies the path to the Certification Authority-issued certificate of the server to which you are connecting.

This can be used to verify that you have connected to the correct server.

An error is returned if the certificate does not match the server.

All certificates must be in PEM (Privacy Enhanced Mail) format.

SSL-CERTIFICATE

Specifies the path to a client certificate.

This is used by the server to authenticate this client.

All certificates must be in PEM (Privacy Enhanced Mail) format.

SSL-KEY

Specifies the path to your private key.

If required, the password for your private key can be specified using the SSL-KEY-PASSWORD clause.

All certificates must be in PEM (Privacy Enhanced Mail) format.

ALLOW-REDIRECTION

Specifies that URL redirection or forwarding is allowed.

SFTP-COMMAND

Specifies the SFTP commands to run before the file transfer.

Multiple SFTP-COMMAND clauses can be specified.

The commands supported are those defined by libcurl. For more information, see online help for CURLOPT_QUOTE.

SFTP-POST-COMMAND

Specifies the SFTP commands to run after the file transfer.

Multiple SFTP-POST-COMMAND clauses can be specified.

SFTP-CREATE-DIRS

Specifies that directories in the SFTP file path should be created if not present.

URL-USER

This clause allows for the specification of the user and password to be used for the connection.

URL-LOGIN-OPTIONS

This clause allows a login options string to be specified in accordance with RFC 2384 and RFC 5092.

ON ERROR

Perform these statements if the URL call was unsuccessful.

If the ELSE clause is included, the procedural statements following the ELSE are performed if the operation is successful.

The ELSE section of an ON ERROR is only performed if there is no error.

Notes

  • This statement is supported only for runtime release 7.1 or higher.

  • The CALL-URL statement passes unknown methods as a custom request.

  • The CALL-URL statement can specify an unknown method with a http body.

  • The CALL-URL statement supports the SFTP protocol. This can be specified as the protocol within the URL. The URL also specifies the path to the remote file (for example, "sftp://remotehost/mydir/myfile");. A HTTP-METHOD of "upload" must be specified to send a file.

Example

call-url  "https://www.myhost.com/myapi?myparam=123"
   http-body is my-xml-document-blob
     http-method is "post"
   returning
     http-body in file "/tmp/response.xml"
