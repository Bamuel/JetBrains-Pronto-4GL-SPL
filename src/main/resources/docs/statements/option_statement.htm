OPTION statement

Displays an entry in a menu, or a button on the screen.

A selection (option) in a menu is implemented using the first syntax variant below. When the option is selected by the user, all procedural statements following this statement, up to the next OPTION statement or the end of the MENU, are performed.

A button (option) on the screen is implemented using the second syntax variant below. This variant is valid in the DETAIL section of a FORM-ENTRY screen; it is currently not supported in data grids or review screens. When the option is selected by the user, all procedural statements before the END-OPTION clause are performed; no other procedural code in the screen is executed, however.

In the Pronto Xi web interface, the tree menu refers to the left hand side standard menu.

Syntax

option alpha-expression @row,col [to @row,col] //menu syntax
   [rows numeric-expression]
   [columns numeric-expression]
   [security [level] [is] number]
   [when condition]
   [help [is] expression]
   [log [literal-string]]
   [scale [is] expression]
   [colour [is] colour [on colour]]
   [bold] [italic] [underlined]
   [bitmap [is] expression
   [bitmap-pushed [is] expression]
   [bitmap-focus [is] expression]]
   [bitmap-hover [is] expression]
   [bitmap-greyed [is] expression]
   [no-aspect-ratio]
   [icon [is] icon-name]
   [button-style [is] button-style] 
   [default-button [is] expression]
   [text-position [is] expression]
   [hot-key [is] expression]
   [hidden]
   [absolute-coordinates]
   [no-theme]
            
option //button syntax 
   standard-option-clauses
   [detail]
   procedural-statements
end-option

Clauses

Clause Description

@row,col

Screen coordinates to position option description. The row and column values can be arithmetic expressions.

Valid (and unique) screen coordinates are currently required even if the menu is displayed by default using a tree menu or popup menu.

Decimal places are supported in the coordinates.

@10.5,19.2

While decimal places are supported, coordinate addressing is still based on a character grid. Unless otherwise specified, when an area of the screen is addressed, that area includes a full character cell position to the bottom right of the end coordinate.

display ws-text @10,10 pic x(20)

The above example occupies an area of the screen from @10.0,10.0 to @10.999,30.999.

display ws-text @10.5,10.5
    pic x(20)

The above example occupies an area of the screen from @10.5,10.5 to @11.4999,31.4999.

display bitmap "abcd.bmp" @10,10
    to @14,20

The above example occupies an area of the screen from @10,10 to @14.9999,20.9999

window @1,1 to @23,80

The above example refers to an area from @1.0,1.0 to @23.999,80.999.

The coordinate for the origin of the screen (that is, top/left) is @1,1 not @0,0.

The REFRESH statement can be used to update the display after changing the width or height of an OPTION button.

If either of the row or column coordinates are negative, for example @-1,0, the button is hidden from the screen. If the button had previously been shown, it is removed after the screen is re-displayed using the REFRESH statement.

TO @row,col

Secondary coordinates for the menu option’s button.

This is the coordinate of the opposite corner of the button. This allows a button based menu option to extend over more than a single line.

This option is ignored when a tree menu or popup menu is used.

ROWS

COLUMNS

In responsive screens, the ROWS and COLUMNS clauses allow you to specify the size of a button, instead of using coordinates.

The rows/columns values are based on the traditional character grid for the screen.

option "All Sites" columns 10 rows 1

SECURITY

Specifies the security level for the option.

Only users with a security level of at least this are allowed to select this option. This applies to the user’s global security level number.

Application security does not use the global security.

WHEN

This option is only allowed when the condition specified evaluates to TRUE.

HELP

Defines a help line for the menu selection or button defined by this statement.

LOG

Log the selection of this menu option. This writes an entry to the menu log file (menu.log) contained in the Pronto Xi lib directory.

If literal-string is specified, this is used as the description of the menu option, otherwise the menu option prompt is used.

SCALE

Scaling factor for the text’s font size when displayed in a button based menu. The scaling factor is specified as a percentage (100 being normal font size).

Text with a scaling factor is vertically centred and will not wrap if larger than the width of the button. Text without a scaling factor is not centred, but will wrap in the bounds of the button.

The REFRESH statement can be used to update the display after changing the scale of an OPTION button.

COLOUR or COLOR

Usage examples:

colour is red on black
colour is blue
colour is red on 0

When 0 or no colour is specified, as in the third example above, the default foreground or background colour set by the user in the colour settings of the Pronto Xi Client is used.

Microsoft Windows RGB colour values can be specified by using the RGB-TO-COLOUR() function.

Either the foreground colour only, or both the foreground and background colours can be specified. The colours can be specified as an expression (and can be varied at runtime). Fields storing colours must be at least PIC 9(8).

A field type of COLOUR (for example, TYPE IS COLOUR) can be used.

The COLOUR clause does not affect tree or drop down menu items. It also has no effect if the buttons are drawn with the XP button style since the style dictates the button colour. To prevent the XP button theme from being used, include the NO-THEME clause.

BOLD

ITALIC

UNDERLINED

Bold, italic and underline can be used to change the way text is displayed in the button. If the UNDERLINE attribute is used on an OPTION statement that does not specify a bitmap, the button is show in a hyperlink style.

BITMAP

BITMAP-PUSHED

BITMAP-FOCUS

BITMAP-HOVER

BITMAP-GREYED

NO-ASPECT-RATIO

Specifies a bitmap to be used for the OPTION statement when shown in a button-based menu. A prompt must still be specified for the OPTION statement, but this is only used on a character-based terminal.

There are three additional bitmaps that can be specified:

  • BITMAP-PUSHED - the bitmap to display when the button is pushed/pressed.

  • BITMAP-FOCUS - the bitmap to display when the focus is on the button (that is, it is the current button).

  • BITMAP-HOVER - the bitmap to display when the mouse pointer is hovering (or floating) over the button.

  • BITMAP-GREYED – the bitmap to display when the underlying data is non-existent and therefore the button is unavailable (typically used with a WHEN statement).

The BITMAP clause can be specified on its own, but it is advisable to include the others as well. All four bitmaps must be the same size. Only Microsoft -supported bitmap formats (for example, BMP and JPEG) are currently supported.

The bitmaps specified for the OPTION statement are considered part of the application software. They are searched for in the same manner as any application program. They should normally be located in one of the directories specified in the PROPATH environment list. If they are not in one of the PROPATH directories a full path name to the bitmap file must be specified in order for it to be found.

If a single coordinate is specified, this coordinate refers to the location of the top left corner of the bitmap. If a TO coordinate is specified, the bitmap is scaled down (if necessary) to fit in the specified area.

If the bitmap is smaller than the area, it is stretched to fit in the area specified for the bitmap. The aspect ratio of the bitmap is preserved if any scaling is performed, unless the NO-ASPECT-RATIO clause is included.

The NO-ASPECT-RATIO clause specifies that the aspect ratio of the bitmap is not to be maintained if the bitmap is stretched or shrunk. That is, the bitmap is stretched or shrunk both vertically and horizontally to fully fit the specified area.

option "A/R" @3,20
    help is "The Accounts Receivable module"
    bitmap is "deb/deb.bmp"
    bitmap-pushed is "deb/debp.bmp"
    bitmap-focus is "deb/debf.bmp"
    bitmap-hover is "deb/debh.bmp"

option "A/R" @3,20
    when ok-to-display
    bitmap is "deb/deb.bmp"
    bitmap-greyed is "deb/debg.bmp"

ICON

The icon used in screen and tree menu based buttons can be specified using the ICON clause on the OPTION statement.

The following predefined icon types are available:

  • FOLDER-ICON

  • ADMIN-FOLDER-ICON

  • REPORT-FOLDER-ICON

  • ADMIN-ICON

  • REPORT-ICON

  • SCREEN-ICON

  • TRANSACTION-ICON

  • PERIOD-END-ICON

  • QUESTION-ICON

  • BLANK-ICON

  • FAVOURITES-FOLDER-ICON

  • LOCKED-FOLDER-ICON

  • BLUE-FOLDER-ICON

  • RED-FOLDER-ICON

  • GREEN-FOLDER-ICON

  • ENQUIRY-FOLDER-ICON

    This icon should not be used in the Pronto Xi Enterprise Client. It should only be used in the Pronto Xi web interface to indicate a file upload option.

Icons are not used in the Pronto Xi web interface menu.

option "TESTING" @10,10
    icon is screen-icon

The OPTION statement allows the icon type to be specified by a numeric expression (or field name) to allow for variable options. The MODE declaration, however, does not allow expressions.

set option-title = "TESTING"
set icon-number = screen-icon
...
option option-title @10,10
    icon is icon-number

An extra two characters are added to the width of the button to allow for the size of the icon if the width is not specified.

BUTTON-STYLE

Allows the program to dynamically control the style and some highlight attributes of the button.

The button styles are specified as one of the following defined values:

  • RADIO_BUTTON_STYLE

  • RADIO_BUTTON_SET_STYLE

  • CHECK_BOX_STYLE

  • CHECK_BOX_SET_STYLE

  • TAB_BUTTON_STYLE

  • ACTIVE_TAB_BUTTON_STYLE

  • NO_BORDER_STYLE

  • STANDARD_BUTTON_STYLE

    In addition to button styles, the following highlight attributes can also be used:

  • UNDERLINE_STYLE

  • BOLD_STYLE

  • ITALIC_STYLE

Multiple highlight attributes can be specified.

set my-button-style = check_box_style + 
    bold_style + italic_style
...
option "Test Button" @10,10 
    button-style is my-button-style

The STANDARD_BUTTON_STYLE is required if UNDERLINE_STYLE is specified because underline by itself is a borderless hyperlink button style.

Tab buttons are intended to overlay the top of a BOX by a single pixel. They will inherit the internal colour of the box overlaid.

box @11 - (1 / get-system-metrics(2)),2
    to @23,40 absolute-coordinates
...
option "Tab 1" @10,10 to @10,15
    button-style is tab1-style

A REFRESH statement must be issued to refresh the displayed styles of a button when they change.

Pronto Xi Thin Client 6.7v0.5 or higher is required.

DEFAULT-BUTTON

Allows the specification of the default button for traditional button based menu. The clause has no effect if the menu is shown in a tree or popup format. The clause requires a conditional Boolean numeric expression to indicate either TRUE or FALSE. If multiple OPTION statements have a DEFAULT-BUTTON clause which evaluates to TRUE, the first one identified is used.

option "Testing" @10,10 default-button is TRUE

TEXT-POSITION

Allows the location of text in the button to be specified for the Option.

If the BITMAP clause is also used, a TEXT-POSITION value greater than zero indicates that the text for the Option is to be overlaid on top of the bitmap, instead of being ignored.

The position for the text is specified directly after this clause as a number between 1 to 9 or 11 to 19. The numbers are based on the following grids:

1

2

3

11

12

13

4

5

6

Or

14

Where 1:11 means top/left; 2:12 means top/centre and so on.

Position numbers between 11 and 19 indicate that the text is to be displayed in multiple lines. The text must specify the line breaks using <br> tags.

option "This is<br>a three line<br>button"
       @10,10 to @14,30 text-position is 15

HOT-KEY

Used to specify a hot key that is to be associated with the option. When the hot key is pressed the OPTION is activated (depending on any WHEN condition).

Currently only function keys (F1 - F24) can be specified as hot keys.

The numeric codes to identify the function keys are the same as the function key number (that is, the code for F5 is 5).

hot-key is 5

Hot keys are not processed in popup menus.

HIDDEN

Used to indicate the option is hidden (non-display).

The primary intention for this clause is for use with the HOT-KEY clause to enable hot key functions without displayed a button.

Co-ordinates are optional if HIDDEN is specified, however, the option text is still required.

ABSOLUTE-COORDINATES

Affects the interpretation of the TO coordinate, if specified. The area specified does not include any part of the screen below or to the right of the TO coordinate. That is, it does not allow for the additional character position.

display bitmap "abcd.bmp" @10,10 to @14,20
    absolute-coordinates

The above example occupies an area of the screen from @10,10 to @14,20 only.

NO-THEME

Used to disable the drawing of the button using the defined XP theme (that is, this clause will draw the button using a classic button style).

standard-option-clauses

One or more of the clauses (from SECURITY through to NO-THEME) in the first syntax variant of this statement.

Notes (first syntax variant)

  • The character required to select the OPTION can be specified by prefixing the character with an ampersand (&) in the prompt (for example, E&xit specifies x).

  • To print a single ampersand a double ampersand (&&) must be specified in the prompt.

  • If the prompt does not specify a selection character explicitly, the first printable character in the prompt is used.

Notes (second syntax variant)

  • Although the OPTION statement is specified in the DETAIL section of a screen, it does not form part of the logic of the DETAIL section and therefore will not affect the logic flow of the DETAIL section.

  • The WHEN clause of the OPTION statement should be used to flag when the option is valid.

  • This statement can be used to implement button logic that is available when the user is either at mode selection or in Entry/Correct mode. The END-OPTION statement is needed to identify the end of the statements associated with the OPTION statement.

  • There is currently no way to select or navigate to an OPTION button using a keyboard. Only mouse control is currently implemented.
