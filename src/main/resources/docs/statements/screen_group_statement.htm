SCREEN-GROUP statement

A screen group marks the start or finish of a screen group within a responsive screen.

In a fixed position screen (non-responsive), information is grouped together on the screen using [@row,col] to define a set position for the information and box [@row,col to @row,col] [title is expression] to indicate the group.

In a responsive screen, information is grouped together with an explicit link called a SCREEN-GROUP. All information contained in a screen group is displayed vertically within that group, unless specified to do otherwise. When a group moves horizontally or vertically, the information within that group will move with the group and re-position itself automatically.

Syntax

screen-group
   [when condition]
   [title [is] alpha-expression]
   [next row]
   [horizontal]
   [width [is] expression]
   [group-style [is] numeric-expression]
   [tag [is] numeric-literal]
      procedural-statements
end-screen-group

Clauses

Clause Description

WHEN

Disable a screen group.

The WHEN clause is processed once only upon entry to the procedure.

When the controlling screen is created, the group and its contents are disabled (that is, not displayed) if the condition specified is FALSE.

Any procedural code within a disabled group is processed as normal.

An ACCEPT in a disabled group is processed as if there was a failed WHEN clause on the ACCEPT.

A REFRESH DATA or REFRESH WHENS statement re-evaluates all WHEN conditions for any screen groups that are defined. Groups are disabled or displayed as required. For more information, see REFRESH statement.

TITLE

The TITLE clause is available for any SCREEN-GROUP.

This clause sets the title on the title bar of the screen group.

It is processed once only upon entry to the procedure.

The PROGROUPTAGERROR environment variable can be used to control whether an error or warning is displayed if a SCREEN-GROUP does not contain a TITLE clause or TAG clause. A title or tag is required to uniquely identify the SCREEN-GROUP for customisation with Screen Customiser. For more information, see PROGROUPTAGERROR environment variable.

NEXT ROW

By default, each screen group attempts to position itself horizontally to the right of the previous screen group.

If there is not enough space to position the screen group horizontally, it is shifted vertically.

The NEXT ROW clause is used to force the screen position and screen group vertically in relation to the previous group.

HORIZONTAL

All information contained in a screen group is displayed vertically within that group, unless specified to do otherwise.

Use the HORIZONTAL clause with a screen group to make all the information display horizontally within that group, if enough space is available.

If there is not enough space to position the fields horizontally, they are shifted vertically.

screen-group
   horizontal
   title "General"

WIDTH

This should be used rarely and for specific purposes since it is contrary to the concept of a responsive screen.

Each screen group automatically takes up one third of the screen size if enough space is available, and if no override width is defined.

Use the WIDTH clause to increase the horizontal size of any single screen group.

The WIDTH clause can be defined between 1 and 12.

By default, the width of any screen group is 4 and does not need to be specified.

By defining the width, you can make a single screen group take up the horizontal space of a proportion of the screen.

That is:

  • WIDTH 12 can take up all of the screen's horizontal space

  • WIDTH 6 can take up half of the screen's horizontal space

  • WIDTH 8 can take up two thirds of the screen's horizontal space

  • WIDTH 9 can take up three quarters of the screen's horizontal space

  • and so on.

The horizontal space of the screen is relative to its breakpoint.

The width for a screen group is different than for a drop-down. In a screen group, the width is a proportion of the horizontal space of the screen. In a drop-down, the width is specified in characters.

For example, you might specify a width of 12 or 6 where you have very large fields in a screen group and would like to provide additional space.

GROUP-STYLE

The visual layout of a screen group is controlled by its style.

The style controls screen group properties such as tile size, indenting, and so on.

The style values supported are defined by the Pronto Xi web interface, and are limited.

There is no need to specify the style unless there is a requirement to use something other than the standard style.

TAG

Uniquely identify a screen group for screen customisation purposes, if there is no unique title for the screen group.

The first screen group defined will have a default tag value of 9999 assigned if it does not define a TITLE clause or TAG clause.

Screen groups in a responsive screen might have a title, which means that they don't necessarily need a tag to be unique. However, sometimes a group is used for screen alignment purposes, for example, so might or might not have a title. In this case, it is important for the screen group to have a unique tag.

A literal number between 1 and 65535 must be specified for this clause. Expressions are not allowed.

Screen Customiser creates *.pcs text files which contain instructions for the runtime. These instructions are simple and identify screen elements by name. If there are no names or the names are not unique, then it is not possible for the runtime to identify the element in order to carry out the instruction.

Mostly, the names relate to titles and field names. However, some screen elements are not directly related to titles and field names, can be repeated within a procedure, and some titles are optional. Therefore, tags can be defined to provide a unique identifier to the screen element.

A tag needs to be unique within the screen in which it occurs. It does not have to be unique within the entire program, unless it is in a procedure, not a screen. Personalisation settings are screen specific, but consider all non-screen procedures to be screen main.

screen control-screen
   local
      lf-credit-limit-desc   pic x(20)
      lf-credit-limit-available   like so-order-total-amount
   //
   title 'Customer'
   responsive
   select * from deb-master name-and-address-master
      where na-type = "C"
   window-position 5
   start-on-current-record
   allowed search correct
before
detail
   do get-available-credit returning lf-credit-limit-available lf-credit-limit-desc
   //
   screen-group
      title "Detail"
      field-group
         tag 2
         accept accountcode
            when screenmode() = search
            show-value
         display shortname
      end-field-group
   end-screen-group
   //
   screen-group
      tag 3
      accept dr-industry-code
      drop-down width 40 concat-title
         select sys-tbl-code sys-description from system-table where sys-tbl-type = "CI"
      end-drop-down
      accept dr-industry-sub-group
      drop-down width 40 concat-title
         select sys-tbl-code sys-description from system-table where sys-tbl-type = "CG"
      end-drop-down
   end-screen-group
   //
   screen-group
      field-group
         width 40
         accept dr-credit-limit-amount  pic zzz,zzz,zzz,zz9.99
         option 'Credit Limit Balances' rows 1 columns 23
            do gen-enquire
         endoption
      end-field-group
      //
      field-group
         width 40
         display lf-credit-limit-available pic ---,---,---,--9.99
            title 'Available Credit'
         display lf-credit-limit-desc pic x(23)
            title 'Credit Status'
      end-field-group
   end-screen-group
   //
   screen-group
      title "Contacts"
      field-group
         tag 5
         accept na-country-code pic x(4)
         accept na-phone pic x(32)
      end-field-group
      //
      field-group
         tag 6
         display na-country-code pic x(4)
         accept na-fax-no pic x(32)
      end-field-group
   end-screen-group
   //
   confirm auto
   confirmed
   endconfirm
endscreen//control-screen --------------------------------------------------
