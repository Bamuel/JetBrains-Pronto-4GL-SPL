COMMAND statement

Executes the specified operating system command or program.

This statement can be used to run any executable on the system. In UNIX/Linux, a shell script can be run as long as execute permission is allowed.

This statement can also be used to execute/open a Microsoft Windows-based document using standard Microsoft Windows functionality (for example, open an Microsoft Excel spreadsheet or a Microsoft Word document).

Syntax

command program-name
   [parameter[s] [is|are] alpha-expression ...]
   [home [is] alpha-expression]
   [no-message]
   [no-refresh]
   [no-xterm]
   [max-parameters]
   [batch]
   [background|no-wait]
   [wait]
   [external]
   [on error]
      procedural-statements
    [else
      procedural-statements]
     end-on]

Clauses

Clause Description

program-name

The name of the program to execute.

It can be any valid alphanumeric expression.

PARAMETERS

List of parameters to pass to the program.

Only alphanumeric parameters can be passed. Arrays are supported.

On Microsoft Windows it is often best to add COMMAND statement parameters as individual expressions enclosed in surrounding quotes as illustrated in the following examples:

command strconcat(getenv("PRONTO"),"\Util\RUNCMD.exe")
    parameters "/c" 'dir' 'c:\tmp\*.txt' '>' "C:\tmp\list.out"

command strconcat(getenv("PRONTO"),"\Util\RUNCMD.exe")
    parameters "/c dir c:\tmp\*.txt > C:\tmp\list.out"

When using pipe directives, they too must be in separate quotes.

HOME

Execute the command with this home directory.

NO-MESSAGE

Suppresses the display of error messages, for example messages that indicate that the called program failed with a signal.

NO-REFRESH

Stops the COMMAND statement from refreshing the screen when control is passed back to the calling program. This should only be used when there is no possibility of the child program writing to the screen, as subsequent screen output from the calling program can be incorrectly positioned.

This clause is obsolete as it applies to character based terminals only.

NO-XTERM

Stops the COMMAND statement from calling up an xterm session to run the child task when run on an X terminal. This should only be used when the child task being called is an X-Windows program.

This clause only has an effect when running on the X-Windows version of Pronto Xi.

MAX-PARAMETERS

Allows a numeric expression to be specified which evaluates to the maximum number of parameters to be passed, regardless of the number of parameters specified.

BATCH

Submit this command to queue zero of the Pronto Xi batch queue. The program will not wait for the command to be executed.

BACKGROUND

or NO-WAIT

Flags that the child program should be detached from the calling program and run in the background on graphical systems (for example, Microsoft Windows and X-Windows). The calling program will return immediately, and will not wait for the child to complete. The child program will run independently of the calling program.

The EXIT-STATUS function will only indicate the success of the command, and not the actual exit status of the child program.

This clause only affects Windows based systems.

WAIT

Wait for child program to finish before continuing.

This is the default, although this might not be possible when opening Windows documents.

EXTERNAL

Run on the local machine (workstation).

The user must be running the Pronto Xi Client on their workstation when using a UNIX based server. This clause is ignored when the application is run on a Microsoft Windows-based system (as you are already running on the workstation), for example:

if operating-system() != "WINDOWS"
    command 'send2proterm' parameters '-r'
       ws-unix-file-name ws-file-name
    on error
        message "a c:\tmp must exist"
    end-on
endif
command ws-file-name external no-wait
  on error
    message "Could not open", ws-file-name
  end-on

ON ERROR

Perform these statements if the command specified could not be executed.

If the ELSE clause is included, the procedural statements following the ELSE are performed if the operation was successful.

Any invalid home directories are trapped.

The ELSE section of an ON ERROR is only performed if there is no error.

Notes

  • The PARAMETERS clause of the CALL/QUERY/COMMAND statements allow numeric values to be specified. Parameters for external programs are still passed in alphanumeric format; however, numeric values are automatically converted into their preferred alphanumeric format when passed.

  • To open a Microsoft Windows-based document, the required software for that document type must be installed on the workstation. For example, to open a Microsoft Excel spreadsheet, Microsoft Excel must be installed on the workstation.
