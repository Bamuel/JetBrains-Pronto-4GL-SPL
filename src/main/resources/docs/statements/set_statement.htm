SET statement

Assigns a value to a field in a specified object, or declared in a FIELD statement.

Syntax

set field-name assignment-operator expression [no-warning]
set array-name[*] = expression [no-warning]
set object-name.* = object-name.*
set object-name.* = *
set * = object-name.*
set page-no|default font assignment-operator expression
   [on report-name]

Clauses

Clause Description

assignment-operator

The following assignment operators are supported:

= += -= *= /= %=

The combination assignment operators are a shorthand way of expressing a calculation that involves the field being SET.

+= means add the value of the expression to the field.

The following two SET statements are equivalent:

set a = a + (5 * b)
set a += 5 * b

expression

Any valid alphanumeric or numeric expression. The type of expression (numeric or alphanumeric) must match that of the field. Date and time formats are considered numeric, and can have arithmetic performed on them (for example, adding 1 to the date).

NO-WARNING

If the environment variable PROCMPTRUNCWARN is set to Y, the procmp compiler displays a warning when it detects that a larger alpha field is being assigned to a smaller alpha field in the SET statement. This is to help identify the potential truncation of multi-byte characters, for example UTF-8 characters.

This check can be suppressed in individual cases by adding a NO-WARNING clause to the SET statement.

The NO-WARNING clause can be specified anywhere within the statement to suppress any subsequent warning message generated for the statement.

array-name[*]

The SET statement allows the specification of an asterisk (*) as the subscript to the field being set. This implies that every element of the array is to be set to the specified value.

set abc-array[*] = 0
set xyz-array[*] = "ABCD"

This is the only place where an asterisk (*) can be specified as a subscript.

object-name.*

This allows objects defined with the NO-JOIN clause to move field values between other NO-JOIN objects containing some or all of the same fields, and between the global fields of the same name (using * by itself).

PAGE-NO

Allow the value returned by the PAGE-NO() function for a report to be changed.

DEFAULT FONT

Allows the default font value for a report to be changed.

ON

If PAGENO or DEFAULT values are being changed, this clause is allowed to define which report is being referred to. Report-name must be a valid report name defined in the REPORT statement.

DATE-TIME

When initialising the date or time related fields, instead of setting them to a literal 0 (Zero) use the following appropriate date/time type:

set lf-date = zero-date
set lf-time = zero-time
set lf-date-time = zero-date-time

Notes

  • Control characters (ASCII codes 1 to 31) can be stored in alphanumeric fields, although their use is discouraged.

  • Fields can also be set using the return values of a procedure.
