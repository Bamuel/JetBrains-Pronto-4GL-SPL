CONFIRM statement

Prompts for user confirmation in a screen, procedure or menu.

If confirmation is not given, the escape flag is set (equivalent to backing out at an ACCEPT) and the action specified (for example, record removal) is not performed.

Only one CONFIRM statement can be present in any SCREEN, PROCEDURE or MENU. It must be the last statement when used in the DETAIL section of a SCREEN or MENU.

CONFIRM cannot be included in conditional code, that is, in a FOR, WHILE or REPEAT loop, or in an IF statement.

Syntax

confirm [@row,col]
   [auto [when condition]
   [prompt [is] alpha-expression]
   [no-prompt]
   [default [is] yes|no]
   [no-window|window]
   [before 
      procedural-statements]
   [confirmed 
      procedural-statements]
   [not-confirmed 
      procedural-statements]
   [after 
      procedural-statements]
end-confirm

Clauses

Clause Description

@row,col

Confirmation is requested by means of a popup input field at the specified coordinates. The row and column values can be arithmetic expressions.

Decimal places are supported in the coordinates:

@10.5,19.2

Row/column values less than 1.0 are invalid.

The coordinate for the origin of the screen (that is, top/left) is @1,1 not @0,0.

While decimal places are supported, coordinate addressing is still based on a character grid. Unless otherwise specified, when an area of the screen is addressed, that area includes a full character cell position to the bottom right of the end coordinate:

display ws-text @10,10 pic x(20)

The above example occupies an area of the screen from @10.0,10.0 to @10.999,30.999.

display ws-text @10.5,10.5 pic x(20)

The above example occupies an area of the screen from @10.5,10.5 to @11.4999,31.4999.

display bitmap "abcd.bmp" @10,10 to @14,20

The above example occupies an area of the screen from @10,10 to @14.9999,20.9999

window @1,1 to @23,80

The above example refers to an area from @1.0,1.0 to @23.999,80.999.

AUTO

Automatically give confirmation without asking, with the exception of screen-mode () = remove.

WHEN

Only ask for confirmation when the specified condition evaluates to TRUE. When not TRUE, confirmation is given automatically.

PROMPT

Use this prompt instead of the default.

NO-PROMPT

This clause suppresses any prompt message to be printed when asking for confirmation.

A three character window is created to accept the response. The input character is the middle character in the window. A coordinate if specified indicates the position of the input character, instead of the start of the window, as is normally the case.

This clause is obsolete and should not be used.

DEFAULT

Use this response if no response is entered. If this clause is not present, entry of a response is mandatory.

NO-WINDOW

Do not display a menu based confirmation window. Instead, request confirmation using a popup input field.

WINDOW

Request confirmation using a menu based confirmation window. This is the default.

BEFORE

Group of commands to be performed before confirmation is required.

CONFIRMED

Commands to perform if confirmation is given. Any automatic I/O operation for a screen mode (for example, the Correct mode) will have already been performed (and succeeded).

If a BEFORE_OPTION trigger in Screen Customiser returns a non-zero exit status to indicate that the option is not to be performed, the CONFIRMED section of the CONFIRM statement, if present, will also not be performed.

NOT-CONFIRMED

Commands to perform if confirmation is not given.

Commands placed here are also performed when backing out of a screen or when the automatic I/O operation for a screen mode (for example, the Correct mode) fails.

AFTER

Commands to perform before the CONFIRM statement finishes, but after any CONFIRMED or NOT-CONFIRMED sections are processed.

Notes

When the SELECT statement for a screen has a WHEN condition and the screen is in Correct mode, if the user steps through each accept field they will be prompted for confirmation after completing the last field. If the user presses F4 at any time to automatically complete the fields, however, the screen will not stop and ask for confirmation, that is, a yes confirmation is assumed in this case. If you want the screen to also stop for confirmation when F4 is pressed, you need to include the NO-OK-CANCEL clause in the SCREEN declaration.

When a screen is in Correct mode, the way the confirm is presented to the user changes depending on whether the NO-OK-CANCEL clause has been used in the SCREEN declaration:

  • If this clause is not used, OK and Cancel buttons are displayed at the bottom of the screen at all times and the user can click OK or Cancel at any time. Clicking OK is equivalent to pressing F4.

  • If this clause is used, no buttons are displayed at the bottom of the screen and a confirmation dialog with Yes, No and Cancel buttons is displayed after all fields have been completed or F4 is pressed.

For more information about the NO-OK-CANCEL clause, see SCREEN declaration.
