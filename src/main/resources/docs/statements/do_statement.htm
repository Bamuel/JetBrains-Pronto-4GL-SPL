DO statement

Performs the specified SCREEN, PROCEDURE, API or MENU defined in the current program.

Syntax

do screen-name|procedure-name|api-name|menu-name
   [screen-mode [screen-mode ...]]
   [same] 
   [once] 
   [sub-screen]
   [initial-mode [is] screen-mode]
   [parameter[s] [is|are] parameter [parameter ...]]
   [returning return-parameter [return-parameter ...]]
   [next routine]
   [no-wait]
   [no-sections]

where
parameter = expression | object-name.*
return-parameter = field-name | object-name.*

Clauses

The entries marked '*' only apply when calling a screen.

Clause Description

screen-mode *

A mode to display on the called screen. This can be a standard screen mode, for example ENTRY, or a user specified screen mode declared in a MODE statement.

The following rules/restrictions apply:

  • Any modes specified here take precedence over the modes defined for the screen being called.

  • If a screen is called with only one mode, the screen is automatically placed in that mode.

  • If the current screen is in show mode, the called screen will also be in show mode and the screen mode specified (including ENTRY and CORRECT) is ignored.

SAME *

Perform the screen using the same screen modes as the current screen.

ONCE *

Only perform the screen for one iteration of the DETAIL section. For example, only allow for entry once instead of multiple times (as is the default).

SUB-SCREEN *

The screen to perform is to be considered as a sub-screen. If the screen performed is backed out of, it is considered equivalent to backing out of the current screen.

INITIAL-MODE *

Place the screen into this mode initially. When the screen has been processed in this mode once, the list of other available modes is displayed for selection. The specified initial mode does not need to be in the list of other modes to perform.

PARAMETERS

Specify fields, values or expressions that are passed to the routine as parameters (arrays supported). The number and data types of the fields passed must match that of the parameter list of the routine being called. Parameters for a routine can also be passed and returned as a complete row/record for an object.

For more information, see PARAMETER declaration.

RETURNING

Specify a field or fields that are to receive the return values from the routine called.

Arrays are supported.

No-join fields can be specified (that is, a field prefixed with a no-join object name).

NEXT

Specify a routine.

This will call the next component library level of an EXPORT routine. The NEXT is specified immediately after the DO and before the name of the routine. That is,

do next routine

NO-WAIT

The use of this clause is restricted to screens that define their own window. This applies to both the calling screen and the open screen.

Indicates that the DO command should not wait for the open screen to exit.

The following rules / restrictions apply:

  • Once the opened screen has been initialised, control is passed directly back to the DO command and procedural execution continues within the parent (calling) screen.

  • Once open, the child screen is put into hibernation. Control is passed to the child screen when its window is clicked. Control is passed to the parent (calling) screen when its windows is clicked.

  • Multiple NO-WAIT screens can be created concurrently.

  • The DO statement cannot specify a RETURNING clause if NO-WAIT is specified.

  • If the DO statement is called for a NO-WAIT screen that is already present, then the existing NO-WAIT screen will be refreshed rather than a new screen created. If the DO statement specifies a parameter list, then these will be updated in the called screen.

NO-SECTIONS

Indicates that the DO command should ignore screen sections when the screen is created for routines that have them defined.

For more information, see SCREEN-SECTION statement.
