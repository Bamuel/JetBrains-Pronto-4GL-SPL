BOX statement

Displays a box or horizontal/vertical line on the screen.

Syntax

box [@row,col to @row,col]
   [double] [no-cross]
   [title is expression]
   [sunken]
   [colour [is] colour [on colour]]
   [absolute-coordinates]

Clauses

Clause Description

@row,col TO @row,col

Specifies the coordinates of the two opposite corners of the box.

The top left corner of the box is at the coordinates given by the smallest row and smallest column value specified. The bottom right corner of the box is at the coordinates given by the largest row and largest column value.

If the same row is specified in both coordinates, the result is a horizontal line. If the same column is specified in both coordinates, the result is a vertical line.

If no coordinates are specified for the box, the box inherits the coordinates of the current screen (or window). This only applies when running on a (dumb) character based terminal. When running on a graphics display device, this is ignored as all windows contain borders.

The row and column values of both coordinates can be arithmetic expressions.

Decimal places are supported in the coordinates, for example:

@10.5,19.2

Row/column values less than 1.0 are invalid.

The coordinate for the origin of the screen (that is, top/left) is @1,1 not @0,0.

While decimal places are supported, coordinate addressing is still based on a character grid. Unless otherwise specified, when an area of the screen is addressed, that area includes a full character cell position to the bottom right of the end coordinate, for example:

display ws-text @10,10 pic x(20)

The above example occupies an area of the screen from @10.0,10.0 to @10.999,30.999.

display ws-text @10.5,10.5 pic x(20)

The above example occupies an area of the screen from @10.5,10.5 to @11.4999,31.4999.

display bitmap "abcd.bmp" @10,10 to @14,20

The above example occupies an area of the screen from @10,10 to @14.9999,20.9999

window @1,1 to @23,80

The above example refers to an area from @1.0,1.0 to @23.999,80.999.

DOUBLE

Draw the box with double line drawing characters if available.

If there are no double line characters defined for the terminal type in use, single line characters are used instead. This clause is obsolete and only applies to character based terminals.

NO-CROSS

This is used when drawing lines so that the ends form a T instead of a cross if they intersect with another line/box. This clause is obsolete and only applies to character based terminals.

TITLE

If a title is specified for a box, the title is displayed left justified on the top line of the box, for example:

box @5,1 to @10,40 title is "Details"

Any BOX statement that includes a TITLE clause is processed as a group box in terms of colour and borders, regardless of whether the title is blank.

SUNKEN

Boxes with this attribute are drawn with a 3D sunken border.

COLOUR or COLOR

Usage examples:

colour is red on black
colour is blue
colour is red on 0

When 0 or no colour is specified, as in the third example above, the default foreground or background colour set by the user in the colour settings of the Pronto Xi Client is used.

Microsoft Windows RGB colour values can be specified by using the RGB-TO-COLOUR() function.

Either the foreground colour only, or both the foreground and background colours can be specified. The colours can be specified as an expression (and can be varied at runtime).

The foreground colour is the colour of the box border. The background colour is the box fill colour.

For sunken boxes the foreground colour is ignored:

box @10,10 to @15,70 colour is black on white

If the SUNKEN clause is also specified, the foreground colour is ignored, however the box is filled with the specified background colour when a background colour is specified.

If a colour is specified for a box with the TITLE clause, the colour only applies to the specified text and not for the box itself.

ABSOLUTE-COORDINATES

The box is drawn at the specified coordinates.

Normally the coordinates at which the box is drawn are offset by half a standard character. That is, the box is drawn in the middle of the character cells, not on the outer edge. This clause ensures the coordinates specified are not offset by the half character.

This affects the interpretation of the TO coordinate, if specified. The area specified does not include any part of the screen below or to the right of the TO coordinate. That is, it does not allow for the additional character position:

display bitmap "abcd.bmp" @10,10 to @14,20
    absolute-coordinates

The above example occupies an area of the screen from @10,10 to @14,20 only.
