GET statement

Reads a record from the specified object.

Syntax

get [field-list from] object-name
   [next|previous [same|different [field-list]]
   [first|last|current|lookup]
   [lock]
   [on index index]
   [key [is] expression ...]
   [on error [error-number ...]
      procedural-statements
    [else
      procedural-statements]
    end-on]

Clauses

Clause Description

FROM

List of fields to retrieve instead of the entire record. Only the values for the listed fields are set to those of the record read. The values of fields in the record that are not listed are not changed.

If this option is not used, all fields for the record read are retrieved.

NEXT

PREVIOUS

Read sequentially the next/previous record for the file.

The PREVIOUS clause is not supported on all database systems and should be avoided.

SAME

DIFFERENT

If the SAME clause is included, the record accessed will have the current value of the specified fields. If the DIFFERENT clause is included, the record accessed cannot have the current value of the specified fields. If no fields are specified, the current key fields are assumed.

FIRST

Read the first record in the object for the specified index.

LAST

Read the last record in the object for the specified index.

The LAST clause, when used with the KEY IS clause, is designed to operate differently depending on whether or not all key fields are specified for the index. If all key fields are not listed in the KEY IS clause for the specified index, then you will be returned the last record in the list that contains the leading key values. That is, you will get the record with the highest values of the non-specified key fields. If all key fields are listed for the specified index , then you will be returned the last record in the list that contains the specified part of the last key field.

The KEY IS clause can contain values for all of the key fields of the index used only if the last key field is alphanumeric.

If you have records with key values X, X1, X2, X3 and you specified a GET LAST KEY IS value of just X, you would be returned the X3 record. That is, the record with the highest value with the leading part of the specified key field.

CURRENT

Reread the current record.

LOOKUP

Read only to determine if the record exists. This option will not affect the currency or any field values of the object. The ON ERROR clause should be used to determine the existence or non-existence of the record.

A LOOKUP clause is different to a normal GET. If the LOOKUP clause is used, the key fields are not set to the values in the KEY IS clause. If you want the key fields to use the values in the KEY IS clause, you will need to set them separately before the LOOKUP clause.

LOCK

Lock the record read. The record will remain locked until it is explicitly unlocked (using the UNLOCK statement) or it is no longer the current record for the object.

An UPDATE operation will unlock the record unless the LOCK option is specified with it.

For more information on how the runtime handles locked records, see LOCK-METHOD statement.

ON INDEX

Read the record using this index. index can be specified by the index number (for example, 1) or by listing fields (in order) which makes up the key to the required index. If no ON INDEX clause is included, the object is accessed on index 1.

KEY

Specifies the key values to use for random (key) reads. If the LOOKUP clause is not used, the values specified here are moved into the corresponding key fields (even if the GET fails). If the LOOKUP clause is used, the key fields retain their previous values.

If the KEY IS clause does not specify a value for every key field, the first record with the specified key values is returned. That is, the record with the lowest value for the unspecified key fields is returned.

ON ERROR

Specifies procedural statements to perform if the GET fails. A list of specific error numbers to which the ON ERROR clause applies can optionally be specified. The most common conditions that will cause the GET to fail are end of file and record not found.

If the ELSE clause is included, the procedural statements following the ELSE clause are performed if the GET succeeds.

The ON ERROR clause of any I/O operation can have a list of error numbers specified (directly after the ON ERROR). The ON ERROR section is only performed if the error is one of those listed. If the error is not in the list, the program aborts with a full error detail screen.

The ELSE section of an ON ERROR is only performed if there is no error.

Notes

  • Only one of the NEXT, PREVIOUS, FIRST, LAST, CURRENT and KEY clauses can be specified.

  • If none of the above clauses are specified the GET will read the record with the current key values.

  • Records in text files (including External and PRN format files) can be read in reverse order by using the PREVIOUS option on the GET statement.

  • The FIRST and LAST clauses for the GET and POSITION statements are also valid for text files.

The ON INDEX clause

A field list if specified only needs to match the leading fields of a valid index. The index used is the first index found with the specified leading fields.

If a partial field list is used for the ON INDEX clause, the operation will act as though the real index only contained the specified fields.

Values do not need to be specified for each of the fields in the index used. The operation will act as though the real index only contained the leading fields that correspond to the number of specified key values.
