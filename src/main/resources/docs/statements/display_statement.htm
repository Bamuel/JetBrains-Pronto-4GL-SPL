DISPLAY statement

Displays the result of an expression on the screen.

Syntax

display expression
   @row,col
   [col expression]
   [pic picture]
   [title expression]
   [multi-line]
   [rows numeric-expression]
   [columns numeric-expression]
   [blank-when-zero]
   [blank-decimals-when-zero]
   [bold] [dim] [flashing] [inverse] [italic]
   [underlined] [bell]
   [foreground] [background]
   [data] [prompt]
   [colour colour [on colour]]
   [scale expression]
   [fixed-width-font]
   [left|centre|right|centre-coord|right-coord]
   [help-is expression]

display bitmap expression @row,col [to @row,col] no-wait

Clauses

Clause Description

expression

A valid numeric or string expression.

@row,col

The @row,col coordinates do not need to be specified if they are part of a responsive screen.

Screen coordinates at which to display the expression. The row and column values can be arithmetic expressions.

Decimal places are supported in the coordinates:

@10.5,19.2

Row/column values less than 1.0 are invalid.

The coordinate for the origin of the screen (that is, top/left) is @1,1 not @0,0.

While decimal places are supported, coordinate addressing is still based on a character grid. Unless otherwise specified, when an area of the screen is addressed, that area includes a full character cell position to the bottom right of the end coordinate:

display ws-text @10,10 pic x(20)

The above example occupies an area of the screen from @10.0,10.0 to @10.999,30.999.

display ws-text @10.5,10.5
    pic x(20)

The above example occupies an area of the screen from @10.5,10.5 to @11.4999,31.4999.

display bitmap "abcd.bmp" @10,10
    to @14,20

The above example occupies an area of the screen from @10,10 to @14.9999,20.9999

window @1,1 to @23,80

The above example refers to an area from @1.0,1.0 to @23.999,80.999.

COL

Define the data grid column coordinates.

Within an ACCEPT or DISPLAY statement, the column value for a data grid column can be specified using the following syntax instead of @row,col:

[IN] COL[UMN] <expr>

When this syntax is used instead of @row,col, a row value of the window position (normally 1) is used.

display accountcode col 3

Requires runtime 7.6v3.0 and above.

PICTURE or PIC

Display the result of the expression using this format instead of the default format.

A PICTURE on a DISPLAY or ACCEPT adjusts the amount of space used by the field on the screen, but will not limit the amount of characters being displayed (or entered). The amount of characters being displayed (or entered) is based on the definition of the field.

It is possible to display a 30 character field when a PIC X(20) is specified.

The amount of characters displayed will depend upon their size.

The picture string can be specified as an alphanumeric expression that is evaluated at runtime.

See below for a description of valid PICTURE string characters.

TITLE

Review screens and data-grids can set the title for the column using the TITLE clause of the DISPLAY statement (removing the need for separate title displays in the BEFORE section).

If a variable expression is used the value must be set in the BEFORE section or before the SCREEN being opened.

If a DISPLAY or ACCEPT statement in a responsive screen does not specify a title, the title for the field defined in the dictionary will be used.

This applies only when a dictionary field is specified.

The NO-TITLE clause can be specified if no title is wanted.

MULTI-LINE

Defines a free-form text entry field that allows Line Feed (LF) characters to be entered into the data. When using this clause, the text might wrap but there will be no line breaks.

The Pronto Xi web interface interprets this clause as a field that can continue over multiple lines.

This clause can be specified only for responsive screens. The size of the multi-line field is based on the screen group in which it is contained. For more information, see RESPONSIVE clause and SCREEN-GROUP statement.

ROWS

COLUMNS

In responsive screens, the number of rows and/or columns can be specified for the DISPLAY statement.

The rows/columns values are based on the traditional character grid for the screen.

If multiple rows are specified for a DISPLAY, the MULTI-LINE clause must also be specified if line-feeds are allowed to be entered.

display my-long-description rows 4 columns 20

field-group
   display lf-new-multi-line-address
      title 'Address'
      multi-line
      rows lf-max-no-of-addr-lines
      columns 40

BLANK-WHEN-ZERO

BLANK-DECIMALS-WHEN-ZERO

The BLANK-WHEN-ZERO clause generates a PICTURE that displays blanks if the value is zero.

The BLANK-DECIMALS-WHEN-ZERO clause generates a PICTURE that displays blanks where the decimal places would normally be displayed, if the decimal places are zero.

These clauses cannot be used in conjunction with a PICTURE clause.

BOLD

ITALIC

INVERSE

UNDERLINED

DIM

FLASHING

BELL

BOLD, ITALIC and UNDERLINE will apply the appropriate windows font.

DIM is obsolete, but if used is the equivalent of specifying BACKGROUND.

BELL will cause a beep to be sounded.

FLASHING is currently obsolete, but is reserved for future use.

FOREGROUND

BACKGROUND

Specifies whether this DISPLAY should be considered as a title to a data field. BACKGROUND can be used to indicate that the expression is a title to a field. FOREGROUND indicates that the expression is not a title (that is, data). FOREGROUND and BACKGROUND are shown with different colour settings.

BACKGROUND is the default for a DISPLAY in the BEFORE section of a SCREEN or MENU. FOREGROUND is the default in all other circumstances.

DATA

PROMPT

DATA specifies that the expression when displayed is to be enclosed in a data border. PROMPT specifies that no data border should be used.

PROMPT is the default for a DISPLAY in the BEFORE section of a SCREEN or MENU. DATA is the default in all other circumstances.

COLOUR or COLOR

Usage examples:

colour is red on black
colour is blue
colour is red on 0

When 0 or no colour is specified, as in the third example above, the default foreground or background colour set by the user in the colour settings of the Pronto Xi Client is used.

Windows RGB colour values can be specified by using the RGB-TO-COLOUR() function.

Either the foreground colour only, or both the foreground and background colours can be specified. The colours can be specified as an expression, and can be varied at runtime, as shown in the following example.

display lf-positions-taken @1,35 pic z(12)9
   colour is
      ifthenelse(lf-positions-taken > lf-available-positions,black,0)
   on
      ifthenelse(lf-positions-taken > lf-available-positions,red,0)
   title 'Filled Positions'

A field type of COLOUR (for example, TYPE IS COLOUR) can be used. Fields storing colours must be at least PIC 9(8).

When this clause is used in a DATA-GRID, individual cells can have separate colours specified (by using an expression for the colours).

SCALE

Allows the scaling factor of the font used in the DISPLAY statement to be changed. A scale of 100 (that is, 100 per cent) is normal. A scale of 50 is half size and 200 is double, etc. If the BOLD attribute is also specified in the DISPLAY statement, a bold (thick) version of the font with the required scaling is used.

Proportional fonts base their size on their height and not their width, as the width of individual characters varies in the font.

The syntax of the clause is:

display ... scale [is] expression

For example:

display "Hello world" @1,1 scale is 300

FIXED-WIDTH-FONT

This clause causes the font used to be the defined fixed width font, instead of the variable (proportional) width default.

LEFT

CENTRE

RIGHT

CENTRE-COORD

RIGHT-COORD

The LEFT clause causes text to be left justified (this is generally the default).

The CENTRE clause causes text to be centred in the field instead of left justified.

The RIGHT clause causes text to be right justified in the field instead of left justified.

The CENTRE-COORD indicates that the coordinate represents the centre of the field (and also centres the text in the field).

The RIGHT-COORD indicates that the coordinate represents the far right position of the field (and also right justifies the text in the field).

BITMAP

Allows a bitmap image to be displayed on the screen.

The bitmap file is considered as data and must be able to be accessed by either Pronto Xi on the machine that the application is running on, or by Proclient (UNIX versions only).

The Microsoft Windows graphics device interface (GDI) supported list tested for PRONTO 4GL (both Pronto Xi web interface and Pronto Xi Enterprise Client) includes the following bitmap formats: GIF, BMP, JPG, PNG, TIF, and Animated GIF.

If a single coordinate is specified, this coordinate refers to the location of the top left corner of the bitmap. If a TO coordinate is specified, the bitmap is scaled down (if necessary) to fit in the specified area.

If the bitmap is smaller than the area it is stretched to fit in the area specified for the bitmap. The aspect ratio of the bitmap is preserved if any scaling is performed, unless the NO-ASPECT-RATIO clause is included. That is, no TO coordinate is given to specify an area.

The syntax of the DISPLAY BITMAP statement is:

 display bitmap alpha-expression @row,col [to @row,col]

An example statement is:

 display bitmap "c:\bitmaps\bitmap.bmp" @10,10

The BACKGROUND clause can be used to specify that the background bitmaps can be overlaid by text and other fields in a window.

The NO-ASPECT-RATIO clause can be used to indicate the aspect ratio of the bitmap is not to be maintained if the bitmap is stretched or shrunk. That is, the bitmap is stretched or shrunk both vertically and horizontally to fully fit the specified area.

The ABSOLUTE-COORDINATES clause affects the interpretation of the TO coordinate, if specified. The area specified does not include any part of the screen below or to the right of the TO coordinate. That is, it does not allow for the additional character position.

display bitmap "abcd.bmp" @10,10 to @14,20
  absolute-coordinates

The above example occupies an area of the screen from @10,10 to @14,20 only.

NO-WAIT

The NO-WAIT clause can be specified to force a bitmap to be sent immediately to the client, if it is not cached there; otherwise this is not done until user input is requested.

display bitmap "/pronto/bitmaps/test.jpg"
  @10,10 no-wait

HELP-IS

Allows mouse over tool tips to be displayed.

Overflows of displayed fields

Any field or expression outputted using the DISPLAY or PRINT statement that is too large for its default (or specified) PICTURE is displayed as a set of question marks (see later for an exception). A negative value shown with an unsigned picture is also displayed as a set of question marks.

If the value to display is too large for the specified PICTURE, but the whole (integer) portion of the number can be displayed in the space provided by ignoring the decimal places, the value is displayed without decimal places. This means that the value displayed might be out of alignment with other values displayed, however, it does mean that the value is still visible.

Numeric fields whose size when displayed is larger than the allowed number of characters are displayed in exponent format (for example, 1.23E+9) if this fits in the allocated size, instead of the value being shown as all question marks ?. This will result in a loss of precision (that is, rounding); however, it will give an indication of the true value.

A DISPLAY statement that is positioned over a BOX or a BACKGROUND BITMAP is transparent (that is, the box or bitmap will show through). This does not apply to displays that have a background colour specified (for example, the REVERSE or the COLOUR statement).

Numeric picture characters

  • 9

    Specifies a single numeric digit. If this character occurs after the decimal point (if present), it indicates a decimal digit, otherwise it represents an integer digit.

  • Z

    Similar to 9, except leading zeros are suppressed.

  • . (Dot)

    Specifies the location of the decimal point.

  • T

    Specifies the location of the decimal point. This also indicates to show decimal places (including the decimal point) as blank if zero.

  • - (Minus sign)

    Indicates the position of the sign character when a value is negative. If multiple leading minus signs are specified the minus sign will float over leading zeros to be positioned before the first non-zero digit.

  • , (Comma)

    This is used for punctuation in a number. If used with the Z, $ or - pic characters, it is not shown for leading zeros.

  • ( )

    Shows the value as enclosed in brackets when the value is negative. The brackets must surround the other picture string characters.

  • $

    Specifies that the currency sign should be shown. If multiple leading currency signs are specified the currency sign will float over leading zeros to be positioned before the first non-zero digit.

  • DR or CR

    Shows DR or CR if the value is negative. We do not recommend that you use these, as these symbols are meaningless in some countries.

  • %

    Inserts a percent sign.

Picture character repetition

Any single picture character that can repeat (for example, 999999) can be specified in the following format:

picture-char(repetition)

For example:

$(10)9.99-

is equivalent to

$$$$$$$$$$9.99-

Data type

  • DATE-TIME

    The default PICTURE format used (for the DISPLAY, ACCEPT and PRINT statements) is DD-MMM-YYYY HH:MM:SS. The standard DATE and TIME PICTURE characters can be used in combination for the DATE-TIME field. A DATE-TIME field that is directly used in this way will assume that it is based on local time (with no GMT time zone offset). If a DATE-TIME field is stored in GMT, it should not be used directly in a DISPLAY/ACCEPT/PRINT.
