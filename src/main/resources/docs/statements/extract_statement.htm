EXTRACT statement

Performs conditional looping while sequentially reading records from the specified object.

The loop terminates when the object specified reaches the end of file/list.

Syntax

extract object-name
   [all]
   [next|previous [same|different [field-list]]
   [lock]
   [on index index]
   [key [is] expression ...]
   [where condition]
   [finish when condition]
   [on error [error-number ...]  procedural-statements end-on]
   [detail 
      procedural-statements]
   [before  [field-list]
      procedural-statements]
         .
         .
         .
   [after [field-list]
      procedural-statements]
         .
         .
         .
end-extract

Clauses

Clause Description

ALL

Read all records for the specified object. That is, position at the start of the file and read sequentially through. This is the default if there is no KEY, NEXT or PREVIOUS clause.

NEXT

PREVIOUS

Specifies the direction and method of reading records. For more information, see GET statement.

If no direction is specified a default of NEXT is assumed.

The PREVIOUS clause is not supported on all database systems and should be avoided.

SAME

DIFFERENT

If the SAME clause is included, the record accessed will have the current value of the specified fields. If the DIFFERENT clause is included, the record accessed cannot have the current value of the specified fields. If no fields are specified, the current key fields are assumed.

LOCK

Only records that are processed are locked. Records excluded because the WHEN clause evaluates to FALSE are not locked.

If a batched program is locked in, it will continue to retry for a period then give up rather than wait indefinitely. This prevents multiple batched programs from deadlocking.

If a record is unable to lock, the EXTRACT statement crashes.

If you want to detect locked records and skip them, you need to do your own locking.

ON INDEX

Force the EXTRACT to read the object using this index. index can be specified by the index number (for example, 1), or by listing every field (in order) which makes up the key to the required index (see the Notes section below).

If no ON INDEX clause is included, the object is accessed on index 1.

KEY

Specifies a starting position for the extract. If this clause is not specified, the file is automatically positioned at the start (or the end if reading backwards).

WHERE

Extract records only when the condition specified is satisfied, that is, exclude records that do not meet this condition. The verbs WHEN and WHERE are equivalent.

FINISH WHEN

Force the EXTRACT loop to finish when the condition is met.

ON ERROR

Specifies procedural statements to perform if the SELECT fails.

The ON ERROR clause of any I/O operation can have a list of error numbers specified (directly after the ON ERROR). The ON ERROR section is only performed if the error is one of those listed. If the error is not in the list, the program aborts with a full error detail screen.

This clause must be located immediately before the first BEFORE, DETAIL or AFTER clause, and cannot contain an ELSE clause.

You cannot trap EENDFILE as this is not considered an error for the loop.

DETAIL

Specifies procedural code to be performed for every extracted record. Multiple DETAIL procedures are not allowed.

BEFORE

Specifies procedural code to be performed before the DETAIL section when one or more of the specified fields changes its value. Current field values are those of the new record. Multiple BEFORE sections can be used.

A BEFORE section specified without a field list is performed once at the start of the EXTRACT loop if the EXTRACT loop processes records. It is not performed if no records are processed.

AFTER

Specifies procedural code to be performed AFTER the DETAIL procedure when one or more of the specified fields changes its value. Current field values are those of the previous record (that is, they do not match the current record for the object being EXTRACTed). Multiple AFTER clauses can be used.

An AFTER section specified without a field list is performed once at the end of the EXTRACT loop if the EXTRACT loop processes records. It is not performed if no records are processed.

Notes

  • A field list for the ON INDEX only needs to match the leading fields of a valid index. The index used is the first index found with the specified leading fields.

  • If a partial field list is used for the ON INDEX clause, the operation will act as though the real index only contained the specified fields.

  • Values do not need to be specified for each of the fields in the index used. The operation will act as though the real index only contained the leading fields that correspond to the number of key values specified.

  • When the NEXT SAME, FINISH WHEN and WHEN clauses are used for an EXTRACT statement the order of the checks is NEXT SAME, FINISH WHEN then WHEN.

  • Changing a key field in the extract (using an UPDATE statement) and using a NEXT SAME, the key field must be reset back to the old value before continuing the extract, otherwise, the NEXT SAME will fail. There is no need to do a position as the UPDATE statement will not change the position in the table
