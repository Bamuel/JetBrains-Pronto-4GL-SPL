FIELD-GROUP statement

In responsive screens, a field group is used where more than one screen element is considered a single element for practical purposes.

Fields and other displays are automatically displayed horizontally in the order that they are written in the 4GL language, unless instructed to do otherwise.

Therefore, in responsive screen programming, [@row,col] does not need to be specified and can be left to the runtime to position fields.

Using field groups, you can associate multiple fields, description, and drill objects under a single field title.

Syntax

field-group
   [when condition]
   [title [is] expression]
   [vertical]
   [wrap]
   [tag [is] numeric-literal]
      procedural-statements
end-field-group

Clauses

Clause Description

WHEN

The WHEN clause allows field groups to be disabled.

It is processed upon entry to the procedure.

If there is any conditional logic that can be set within the screen that can affect the WHEN clause, then a REFRESH WHENS statement will re-establish the condition in the screen and change the suppression of the field group. For more information, see REFRESH statement.

When the controlling screen is created, the group and its contents are suppressed (that is, not displayed) if the condition specified is FALSE.

Any procedural code within a suppressed group is processed as normal.

An ACCEPT in a suppressed group is processed as if there was a failed WHEN clause on the ACCEPT.

TITLE

The TITLE clause is available for any FIELD-GROUP.

This clause sets the title for the field group.

It is processed once only upon entry to the procedure.

VERTICAL

Use the VERTICAL clause with a field group to make all of the information appear vertically (top to bottom) within that group by default.

field-group
   title 'End'
   vertical
   radio-button mmr-no-end-date
      0 title "End After"
      0 title "End By"
end-field-group

WRAP

Use the WRAP clause with a field group to prevent items that do not fit horizontally from being truncated, and make the group line wrap on to the next row.

field-group
   wrap
   title 'Measuring field group'
   radio-button ws-general-2 
      ' ' title 'Not Applicable'
      '2' title 'Lineal Metre'
      '3' title 'Square Metre'
      '4' title 'Cubic Metre'
      'W' title 'Weighed/Measured'
   default ws-general-2
end-field-group

TAG

Uniquely identify a field group for screen customisation purposes, if there is no unique title for the field group.

A literal number between 1 and 65535 must be specified for this clause. Expressions are not allowed.

Screen Customiser creates *.pcs text files which contain instructions for the runtime. These instructions are simple and identify screen elements by name. If there are no names or the names are not unique, then it is not possible for the runtime to identify the element in order to carry out the instruction.

Mostly, the names relate to titles and field names. However, some screen elements are not directly related to titles and field names, can be repeated within a procedure, and some titles are optional. Therefore, tags can be defined to provide a unique identifier to the screen element.

A tag needs to be unique within the screen in which it occurs. It does not have to be unique within the entire program, unless it is in a procedure, not a screen. Personalisation settings are screen specific, but consider all non-screen procedures to be screen main.

screen control-screen
   local
      lf-credit-limit-desc   pic x(20)
      lf-credit-limit-available   like so-order-total-amount
   //
   title 'Customer'
   responsive
   select * from deb-master name-and-address-master
      where na-type = "C"
   window-position 5
   start-on-current-record
   allowed search correct
before
detail
   do get-available-credit returning lf-credit-limit-available lf-credit-limit-desc
   //
   screen-group
      title "Detail"
      field-group
         tag 2
         accept accountcode
            when screenmode() = search
            show-value
         display shortname
      end-field-group
   end-screen-group
   //
   screen-group
      tag 3
      accept dr-industry-code
      drop-down width 40 concat-title
         select sys-tbl-code sys-description from system-table where sys-tbl-type = "CI"
      end-drop-down
      accept dr-industry-sub-group
      drop-down width 40 concat-title
         select sys-tbl-code sys-description from system-table where sys-tbl-type = "CG"
      end-drop-down
   end-screen-group
   //
   screen-group
      field-group
         width 40
         accept dr-credit-limit-amount  pic zzz,zzz,zzz,zz9.99
         option 'Credit Limit Balances' rows 1 columns 23
            do gen-enquire
         endoption
      end-field-group
      //
      field-group
         width 40
         display lf-credit-limit-available pic ---,---,---,--9.99
            title 'Available Credit'
         display lf-credit-limit-desc pic x(23)
            title 'Credit Status'
      end-field-group
   end-screen-group
   //
   screen-group
      title "Contacts"
      field-group
         tag 5
         accept na-country-code pic x(4)
         accept na-phone pic x(32)
      end-field-group
      //
      field-group
         tag 6
         display na-country-code pic x(4)
         accept na-fax-no pic x(32)
      end-field-group
   end-screen-group
   //
   confirm auto
   confirmed
   endconfirm
endscreen//control-screen --------------------------------------------------
